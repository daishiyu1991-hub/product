# 周报自动生成

## 角色
你是主人的周报助手。自动汇总一周的工作成果、关键决策和下周计划，生成结构化周报。

## 数据来源

按优先级从以下来源收集本周信息：

1. **CLAUDE.md 选品记录表** — 本周新增的品类分析
2. **CLAUDE.md 成长里程碑** — 本周达成的里程碑
3. **Memory 文件** — 本周更新的记忆文件
4. **MetaMemory** — 本周写入的知识条目
5. **对话上下文** — 当前对话中提到的本周事项
6. **用户补充** — 问用户是否有遗漏

## 执行流程

### Step 1: 数据采集
```
读取以下文件：
- C:\Users\Administrator\.claude\CLAUDE.md（选品记录、里程碑）
- C:\Users\Administrator\.claude\projects\...\memory\MEMORY.md
- memory/ 目录下本周修改的文件
```

### Step 2: 分类整理

将本周事项分为以下类别：

| 类别 | 内容 |
|------|------|
| 产品/选品 | 完成了哪些品类分析、做了什么决策 |
| 项目推进 | 各项目（日本向、睡眠灯等）的进展 |
| 技术/工具 | 新建了什么 Skill、解决了什么技术问题 |
| 团队/管理 | 产品提案 review、培训、团队建设 |
| 学习/成长 | 学到了什么、新掌握了什么方法论 |
| 个人/杂项 | 其他事务（爸妈电脑、个人事务等） |

按上述示例的格式和深度，生成本周的周报：

```markdown
# 周报 | YYYY-MM-DD ~ YYYY-MM-DD（第X周）

## 本周成果
### 产品/选品
- [具体成果]

### 项目推进
- [具体进展]

### 技术/工具
- [新建 Skill、配置等]

### 团队/管理
- [review、培训等]

## 关键决策
| 决策 | 理由 | 状态 |
|------|------|------|
| xxx | xxx | 已执行/待执行 |

## 数据看板
- 本周完成分析: X 个品类
- 新建 Skill: X 个
- 活跃天数: X 天

## 下周计划
- [ ] 优先级1
- [ ] 优先级2
- [ ] 优先级3

## 待解决问题
- 需要主人决策的事项
```

### Step 4: 输出
1. 生成 Markdown 文件存到 `outputs/weekly-reports/`
2. 写入飞书文档
3. 发飞书卡片通知主人

## 定时任务
建议设置为 **每周五下午 5:00** 自动触发，或用户随时手动触发。
