# 确定性审计方法论

> 来源：梁宁《产品思维30讲》系统能力 — 确定性
> 核心：用户依赖你，是因为你持续提供确定性。确定性一旦崩塌，用户就会离开。

## 原理

### 什么是确定性？

确定性 = 用户每次使用你的产品，都能得到**可预期的、稳定的**结果。

- **麦当劳的确定性**：无论哪个城市，味道一样 → 用户信任
- **亚马逊产品的确定性**：每个用户收到的产品品质一致 → Review 稳定 → 排名稳定
- **确定性崩塌**：一批次产品出问题 → 差评爆发 → 排名下滑 → 飞轮断裂

### 确定性 vs 创新

| | 确定性 | 创新 |
|--|-------|------|
| 关注点 | 已有的东西不能变差 | 新增的东西要变好 |
| 风险 | 确定性崩塌 = 老客户流失 | 创新失败 = 新客户没来 |
| 优先级 | 先稳住确定性 → 再谈创新 | 确定性是创新的前提 |

### 为什么做确定性审计？

亚马逊产品的死亡螺旋往往不是因为"产品不够好"，而是因为"确定性崩了"：
1. 某批次品控出问题 → 差评增加
2. 差评增加 → 转化率下降
3. 转化率下降 → 排名下滑
4. 排名下滑 → 流量减少
5. 流量减少 → 只剩低价敏感用户 → 更多差评

**这是调节回路（负向飞轮）的典型模式。确定性审计 = 在崩塌前发现裂缝。**

## 四维确定性模型

### 维度1: 品质确定性

> 每个用户收到的产品品质是否一致？

| 审计指标 | 数据来源 | 健康标准 | 警告信号 | 危险信号 |
|---------|---------|---------|---------|---------|
| 星级稳定性 | `product_trend`（近6个月） | 波动<0.2星 | 波动0.2-0.5星 | 连续下降>0.3星 |
| 差评率趋势 | `product_reviews`（Negative） | <15%且稳定 | 15-25%或上升中 | >25%或加速上升 |
| 差评主题集中度 | `product_reviews`（Negative） | 分散（无单一>20%） | 某主题>20% | 某主题>30%（系统性问题） |
| "质量"相关差评占比 | Review中含quality/defect/broke/broken | <10% | 10-20% | >20% |

**品质确定性诊断逻辑**：
```
如果差评主题高度集中（某一问题>30%）→ 系统性品质缺陷 → P0修复
如果差评分散但总量在增加 → 整体品控标准下降 → 检查供应链
如果差评稳定且低 → 品质确定性OK ✅
```

### 维度2: 交付确定性

> 用户下单后是否能按时收到完好的产品？

| 审计指标 | 数据来源 | 健康标准 | 警告信号 | 危险信号 |
|---------|---------|---------|---------|---------|
| 配送相关差评 | Review中含shipping/delivery/package/damaged | <5% | 5-10% | >10% |
| 断货历史 | `product_trend`（SalesVolume） | 无断崖式下跌 | 偶有下跌 | 频繁下跌（周期性断货） |
| FBA/FBM状态 | `product_detail` | FBA | FBM但稳定 | FBM且配送差评多 |
| 库存深度 | 用户提供 | 30-60天 | <30天或>90天 | <14天（随时断货） |

**交付确定性诊断逻辑**：
```
如果销量趋势有断崖式下跌 → 断货 → 检查备货策略
如果配送差评>10% → 包装/物流问题 → 检查包装方案
如果是FBM且配送差评多 → 考虑转FBA
```

### 维度3: 体验确定性

> 用户每次使用产品时，体验是否一致？

| 审计指标 | 数据来源 | 健康标准 | 警告信号 | 危险信号 |
|---------|---------|---------|---------|---------|
| "用了X次后"差评 | Review中含"after X months/weeks/uses" | <5% | 5-15% | >15%（耐久性问题） |
| 使用说明相关差评 | Review中含"instructions/confusing/how to" | <5% | 5-10% | >10%（上手困难） |
| 期望落差差评 | Review中含"not as described/misleading/expected" | <5% | 5-10% | >10%（Listing欺骗） |
| 一致性差评 | Review中含"inconsistent/sometimes works/hit or miss" | <3% | 3-8% | >8%（体验不稳定） |

**体验确定性诊断逻辑**：
```
如果"用了X次后"差评多 → 耐久性/寿命问题 → 材料/工艺需升级
如果"not as described"差评多 → Listing过度承诺 → 调整Listing（而非调产品）
如果"inconsistent"差评多 → 品控一致性问题 → 加强QC
```

### 维度4: 价值确定性

> 用户觉得花的钱值不值？

| 审计指标 | 数据来源 | 健康标准 | 警告信号 | 危险信号 |
|---------|---------|---------|---------|---------|
| 价格相关差评 | Review中含"overpriced/not worth/expensive/cheap" | <5% | 5-10% | >10% |
| 价格趋势 | `product_trend`（Price） | 稳定或微调 | 频繁变价 | 被迫大幅降价 |
| 竞品价格差 | `product_detail` vs 竞品 | 差异<20% | 差异20-40% | 差异>40%（极端溢价或低价） |
| 价值感知词 | Review中含"great value/worth it/love it" | >20%好评含 | 10-20% | <10%（用户不觉得值） |

**价值确定性诊断逻辑**：
```
如果"overpriced"差评多 → 定价过高 or 价值感不够 → 要么降价，要么增加感知价值
如果被迫大幅降价 → 价格战 → 检查是否在别人的价值网里竞争（→ 联动价值网分析）
如果"great value"好评多 → 价值确定性OK ✅，可以考虑微涨价测试
```

## 执行步骤

### Step 1: 数据采集

| 数据 | 工具 | 用途 |
|------|------|------|
| 产品详情 | `product_detail` | 基础信息（价格/星级/Review数/上架时间）|
| 销量趋势 | `product_trend`（SalesVolume） | 识别断货/下滑 |
| 价格趋势 | `product_trend`（Price） | 识别价格波动 |
| 排名趋势 | `product_trend`（Rank） | 识别排名稳定性 |
| 正面评论 | `product_reviews`（Positive） | 确定性亮点（用户觉得什么是稳定的）|
| 负面评论 | `product_reviews`（Negative） | 确定性漏洞（什么在崩塌）|
| 关键词排名 | `competitor_product_keywords` | 搜索确定性（排名是否稳定）|

### Step 2: 分类差评

将负面 Review 按四维确定性分类：

```
品质确定性差评: [X]条 (XX%)
├── [主题1]: X条
├── [主题2]: X条
└── [主题3]: X条

交付确定性差评: [X]条 (XX%)
├── [主题1]: X条
└── [主题2]: X条

体验确定性差评: [X]条 (XX%)
├── [主题1]: X条
└── [主题2]: X条

价值确定性差评: [X]条 (XX%)
├── [主题1]: X条
└── [主题2]: X条
```

### Step 3: 评估每个维度

对每个维度给出状态：

| 状态 | 标准 | 对应颜色 |
|------|------|---------|
| 坚固 | 所有指标在健康范围内 | 🟢 |
| 裂缝 | 有1-2个指标在警告范围 | 🟡 |
| 崩塌 | 有指标在危险范围或多个指标在警告范围 | 🔴 |

### Step 4: 确定性优先级排序

修复优先级 = **崩塌速度 × 影响范围 × 修复难度**

| 优先级 | 标准 |
|--------|------|
| P0 | 正在崩塌且影响所有用户（如系统性品质缺陷）|
| P1 | 有裂缝且在恶化（如差评率趋势性上升）|
| P2 | 有裂缝但稳定（如个别体验问题但不扩散）|
| P3 | 预防性维护（当前OK但有潜在风险）|

## 注意事项

1. **确定性审计是"守"，创新是"攻"** — 先守住再进攻。一个确定性崩塌的产品，再多创新也救不回来
2. **品质确定性是地基** — 如果品质不稳定，其他三个维度都是空中楼阁
3. **交付确定性是隐形杀手** — 很多卖家只关注产品本身，忽略了包装破损、断货对确定性的打击
4. **体验确定性需要长期跟踪** — "用了3个月后坏了"这类差评不会在新品期出现，要持续监控
5. **价值确定性和价格战的关系** — 如果你被迫打价格战，本质上是你的价值确定性在用户心中不够，需要联动**价值网分析**看是否在错误的价值网竞争
6. **自有工厂的优势在品质确定性** — 如果用户有自有工厂，品质确定性应该是最强的维度；如果不是，说明工厂管理有问题
7. **确定性审计 + 飞轮诊断 = 完整体检** — 飞轮告诉你"增长引擎是否在转"，确定性审计告诉你"刹车系统是否可靠"
