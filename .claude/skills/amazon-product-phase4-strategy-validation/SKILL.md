# Phase 4 战略验证（精益开发闭环）

## 定位

`amazon-product-phase4-strategy-validation` 是精益产品开发闭环的第四阶段，衔接上游 Phase 3（MVP 蓝图）和下游 Phase 5（工业设计调研），在投入设计和制造之前，用**三大战略工具**对 MVP 方案进行多维度校验。

核心问题：**这个方案值得投入吗？增长引擎设计好了吗？确定性有保障吗？**

三大工具：
1. **三级火箭** — 产品矩阵规划：不只做一个单品，而是设计从引流到利润的完整产品线
2. **飞轮诊断** — 增长回路设计：验证产品的增长飞轮是否能转起来，识别刹车点
3. **确定性审计** — 稳定性保障：审计品质/交付/体验/价值四维确定性

## 使用方法

```
/amazon-product-phase4-strategy-validation vacuum steamer US
/amazon-product-phase4-strategy-validation vacuum steamer US --source ./path/to/phase3/
/amazon-product-phase4-strategy-validation B0F5PZPHTC US
```

## 执行步骤

当调用此 Skill 时，**先读取本文件，再按需读取 `references/` 下的详细方法论**，执行以下流程：

1. **读取上游数据** — 加载 Phase 3 MVP 蓝图，提取：
   - MVP 功能矩阵（Must-have / Nice-to-have / Cut）
   - 用户画像 + JTBD 任务地图
   - 场景拆解结果 + 价值网定位
   - 成本与盈亏平衡数据
   - 测试标准（Pass/Fail）
   - 供应商沟通指南
   无上游时降级为自行调用 Sorftime MCP 采集数据

2. **竞品数据采集** — 用 Sorftime MCP 采集核心竞品数据：
   - `product_detail` — 竞品基础信息（价格/星级/Review数/上架时间）
   - `product_reviews`（Negative）— 竞品差评（用于确定性审计）
   - `product_trend`（SalesVolume + Price + Rank）— 竞品趋势
   - `product_traffic_terms` — 竞品流量词（用于飞轮诊断）
   - `competitor_product_keywords` — 竞品关键词排名

3. **三级火箭分析** — 读取 `references/three-stage-rocket.md`，设计产品矩阵：
   - **第一级（引流品）**：用什么产品/价格获取首批用户？
   - **第二级（关联品）**：怎么让用户留在品牌生态？配件/耗材/互补品
   - **第三级（利润品）**：利润从哪来？高端型号/品牌溢价/复购
   - 评估工厂能力与产品矩阵的匹配度
   - 输出 12 个月产品上架路线图

4. **飞轮诊断** — 读取 `references/flywheel.md`，设计并验证增长飞轮：
   - 画出产品的增强回路（正向飞轮）：产品力 → Review → 转化率 → 排名 → 流量 → 销量 → 利润
   - 诊断飞轮每个节点的当前状态
   - 识别调节回路（5大刹车）：
     - 🔴 品控刹车（差评导致转化下降）
     - 🟡 竞争刹车（竞品压制排名）
     - 🟡 成本刹车（利润不够支撑广告）
     - 🟡 供应链刹车（断货中断飞轮）
     - 🟡 认知刹车（品类教育不足）
   - 给出加速飞轮 / 松开刹车的具体行动

5. **确定性审计** — 读取 `references/certainty-audit.md`，审计四维确定性：
   - **品质确定性** 🟢/🟡/🔴 — 产品品质是否一致可预期？
   - **交付确定性** 🟢/🟡/🔴 — 库存/物流是否可靠？
   - **体验确定性** 🟢/🟡/🔴 — 用户每次使用体验一致吗？
   - **价值确定性** 🟢/🟡/🔴 — 用户觉得值不值？
   - 按 P0-P3 排序修复优先级
   - 输出确定性防御计划

6. **交叉验证矩阵** — 将三个工具的结论交叉对比：
   - 识别多工具一致指向的核心风险（如品控问题在飞轮和确定性中都出现 = 红旗）
   - 识别工具间的矛盾结论并解析原因
   - 汇总决策依据

7. **战略决策输出** — 综合三大工具结论，给出最终决策：
   - **GO** — 三大工具均绿灯，可进入 Phase 5 工业设计
   - **CONDITIONAL GO** — 有条件通过，列出进入 Phase 5 前必须解决的硬性条件
   - **NO GO** — 建议回退 Phase 1 重新选品，说明原因

8. **输出与验证** — 生成文档二件套（MD + HTML）

## 可视化看板图表要求

HTML 看板中必须包含以下 Section：

### Section 1：战略验证总览
- 三栏卡片：三级火箭概览 | 飞轮状态 | 确定性总评
- 底部大色块显示最终决策：GO(绿) / CONDITIONAL GO(黄) / NO GO(红)

### Section 2：三级火箭
- 火箭图示（三层从下到上）
- 每级的产品/价格/角色/上线时间
- 12个月产品路线图（时间轴）

### Section 3：飞轮诊断
- 飞轮图（环形，每个节点标注状态 🟢/🟡/🔴）
- 5大刹车仪表盘（每个刹车的风险等级）
- 关键行动表

### Section 4：确定性审计
- 四维雷达图（品质/交付/体验/价值）
- 每个维度的指标明细表
- 修复处方优先级表

### Section 5：交叉验证矩阵
- 热力图：工具 × 风险维度
- 一致风险高亮标注
- 最终决策理由汇总

## 上下游对接

- **上游**：Phase 3（MVP 蓝图）— 接收功能矩阵、用户画像、JTBD、场景拆解、价值网定位、成本数据
- **下游**：Phase 5（工业设计调研）→ Phase 6（概念图生成）→ Phase 7（上架复盘）→ Phase 8（迭代）→ Phase 9（规模化）
- 战略验证通过后，将以下信息传递给 Phase 5：
  - 产品矩阵中第一级（引流品）的详细规格
  - 飞轮诊断中的关键差异化要求
  - 确定性审计中的品质标准硬约束

## 数据诚信规则

- **绝不捏造数据**：所有数据必须来自 Sorftime MCP 或明确标注来源
- **区分事实与推测**：推测必须标注 ⚠️
- **交叉验证可信度**：标注每个结论被几个工具支持（1/3, 2/3, 3/3）
