# 五层体验双向工具

## 理论来源

梁宁《产品思维30讲》用户体验模块 — 五层模型：

```
┌─────────────────────────────┐  ← 用户最先感知
│       感知层 (Perception)     │  外观、第一印象、包装
├─────────────────────────────┤
│    角色框架层 (Role Frame)    │  用户是谁、产品为谁存在
├─────────────────────────────┤
│   资源结构层 (Resources)      │  功能、材料、工艺、参数
├─────────────────────────────┤
│    能力圈层 (Capabilities)    │  持续交付能力、供应链、品控
├─────────────────────────────┤
│  战略存在层 (Strategic Why)   │  为什么存在、品牌使命
└─────────────────────────────┘  ← 产品的根基
```

**核心认知**：
- 大多数产品问题出在"用错了层" — 感知层问题用战略层方案解决（重新定位），或战略层问题用感知层方案解决（换主图）
- **诊断方向**（从上到下）：从用户看到的问题出发，逐层深挖到根因
- **设计方向**（从下到上）：从品牌为什么存在出发，逐层向上构建完整体验

## 执行流程

**先读取本文件，再按模式读取 `references/` 下的对应方法论。**

### 模式判断

解析用户参数：
- 第一个参数为 `diagnose` 或 `诊断` → 进入诊断模式
- 第一个参数为 `design` 或 `设计` → 进入设计模式
- 无参数 → 询问用户选择哪种模式
- 如果第一个参数看起来像 ASIN（B0开头10位）→ 自动进入诊断模式
- 其他 → 自动进入设计模式

### 诊断模式执行步骤

读取 `references/diagnosis-mode.md` 后按以下步骤执行：

1. **数据采集** — 调用 Sorftime MCP 获取产品数据
   - `product_detail` → 基础信息（标题、价格、星级、评论数、品牌、卖家）
   - `product_reviews`（Negative）→ 负面体验信号
   - `product_reviews`（Positive）→ 正面体验信号
   - `product_report` → 销量、排名、趋势
   - `competitor_product_keywords` → 关键词曝光能力

2. **五层逐层诊断** — 从感知层到战略存在层，每层评分 1-5 分
   - 每层基于具体数据信号评分（不凭感觉）
   - 识别每层的优势和问题

3. **病根定位** — 找出最弱的层（得分最低）
   - 判断是否存在"改错了层"的问题
   - 分析层间传导关系（底层问题如何影响上层表现）

4. **处方输出** — 针对病根层给出具体优化建议
   - 优化优先级排序
   - 结合峰终定律指导资源分配

5. **输出诊断报告** — 生成结构化报告

### 设计模式执行步骤

读取 `references/design-mode.md` 后按以下步骤执行：

1. **市场速览** — 快速了解目标品类
   - `category_search_from_product_name` → 品类市场规模
   - `keyword_search_results` → 搜索结果现状
   - `product_reviews`（Top 1-2 竞品的 Negative）→ 用户未被满足的需求

2. **战略存在层设计（Why）** — 品牌使命
   - 这个产品解决什么根本问题？
   - 用户为什么需要一个新选择？
   - 你的品牌代表什么价值观？
   - 输出：一句话品牌使命 + 核心价值主张

3. **能力圈层设计（How Sustainably）** — 持续交付
   - 你的工厂/供应链有什么独特能力？
   - 品控如何保证确定性？
   - 售后/客服如何设计？
   - 输出：能力清单 + 确定性承诺

4. **资源结构层设计（What）** — 功能/材料/工艺
   - 基于竞品差评痛点，设计必须有的功能
   - 基于战略存在层的使命，选择匹配的材料/工艺
   - 参数规格定义
   - 输出：MVP 功能清单 + 材料工艺选择 + 参数规格

5. **角色框架层设计（For Whom）** — 目标用户
   - 你的产品把用户定义为什么角色？
   - 这个角色是触达用户的社会角色还是真实自我？
   - Listing 标题/五点如何体现这个角色定位？
   - 输出：用户画像 + 角色定位 + Listing 定位方向

6. **感知层设计（First Impression）** — 第一印象
   - 主图应该传递什么信息？（一眼 get 价值）
   - 包装开箱的峰值体验如何设计？
   - 工业设计的核心差异化在哪？
   - 输出：感知设计要点 + 峰终体验设计

7. **五层一致性检查** — 逐层校验
   - 感知层是否忠实表达了战略存在层的使命？
   - 资源结构层是否支撑了角色框架层的定位？
   - 能力圈层是否能持续交付资源结构层的承诺？
   - 有没有"层间矛盾"？

8. **输出设计蓝图** — 生成结构化蓝图

## 核心输出

### 诊断模式输出

生成 MD 文件：`[日期]_[ASIN]_五层诊断报告.md`

内容包含：
- 产品基础信息卡
- 五层评分雷达图（ASCII 简化版）
- 每层详细诊断（数据依据 + 问题 + 建议）
- 病根分析（最弱层 + 层间传导）
- 优化处方（优先级排序 + 预期效果）
- 峰终体验设计建议

### 设计模式输出

生成 MD 文件：`[日期]_[产品概念]_五层设计蓝图.md`

内容包含：
- 市场速览
- 五层设计方案（从战略存在层到感知层）
- 五层一致性检查表
- Listing 方向指引
- 与 Phase 3-5 的衔接建议

## 与精益开发流程的关系

| 衔接 Phase | 本 Skill 的作用 |
|-----------|----------------|
| Phase 1 之后 | 用**诊断模式**分析 Top 竞品，找到它们的最弱层 = 你的切入点 |
| Phase 3 之中 | 用**设计模式**构建 MVP 的体验架构，与 KANO 矩阵互补 |
| Phase 4-5 之前 | 设计模式的感知层输出 → 指导工业设计方向 |
| Phase 6-7 之中 | 用**诊断模式**分析自己的已上架产品，找到最弱层优先优化 |

## 数据诚信规则

- 所有评分必须基于 Sorftime MCP 数据或明确标注来源
- 推测必须标注 ⚠️
- 诊断模式不凭"感觉"打分，每个分数必须关联具体数据信号
