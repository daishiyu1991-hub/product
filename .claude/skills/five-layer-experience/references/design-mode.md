# 设计模式方法论

> 从品牌的根基出发，逐层向上构建完整体验。方向：战略存在层 → 感知层（从下到上）
> 核心理念："先想清楚为什么存在，再决定长什么样"

## 一、设计模式的独特价值

传统产品开发的问题：
- 先看竞品长什么样 → 模仿 → 加个功能 → 上架 = **没有灵魂的产品**
- 从感知层（外观）开始设计 → 底层空洞 → 层间矛盾 → 用户感知到"不对劲"

五层设计模式的逻辑：
- 先定义灵魂（战略存在层）→ 再建能力 → 再配资源 → 再找人群 → 最后设计外观
- 每一层都服务于下面那层 → 五层一体 → 用户感知到"就是它"

## 二、数据采集（市场速览）

在开始设计之前，快速了解市场现状：

| 步骤 | 工具 | 获取信息 |
|------|------|---------|
| 品类搜索 | `category_search_from_product_name` | 品类规模、均价、竞争度、Top3品牌集中度 |
| 关键词搜索 | `keyword_search_results` | 搜索结果前3页的产品形态和价格带 |
| 竞品差评 | `product_reviews`（Top 1-2竞品 Negative） | 用户未被满足的需求 = 设计机会 |
| 竞品好评 | `product_reviews`（Top 1-2竞品 Positive） | 用户真正在乎什么 = 必须保留 |
| 品类关键词 | `keyword_extends` | 用户搜索意图分布 |

## 三、五层设计问题清单

### 第5层（底层）：战略存在层设计

**核心问题：这个品牌/产品为什么要存在？**

回答以下问题：

| # | 问题 | 输出 |
|---|------|------|
| 5.1 | 这个产品解决的根本问题是什么？（不是功能层面，是人性层面） | 一句话痛点 |
| 5.2 | 如果这个产品消失，谁会最失落？为什么？ | 核心用户画像 |
| 5.3 | 你的品牌代表什么信念/价值观？ | 品牌价值主张 |
| 5.4 | 10年后这个品牌还在做什么？ | 品牌愿景 |
| 5.5 | 用梁宁的情绪分析：你击中的是愉悦、不爽、愤怒还是恐惧？ | 情绪定位 |

**输出格式**：
```
品牌使命：[一句话]
情绪击中：[愉悦/不爽/愤怒/恐惧] — [具体描述]
价值主张：[一句话]
```

**设计辅助问题**：
- 如果你的品牌是一个人，TA 是什么性格？
- 用户使用你的产品时，TA 是在扮演什么角色（社会角色）还是在做真实的自己（自我）？
- **恐惧测试**：用户不用你的产品会怕什么？怕得越深 = 战略存在层越稳

### 第4层：能力圈层设计

**核心问题：你凭什么能持续稳定地交付承诺？**

| # | 问题 | 输出 |
|---|------|------|
| 4.1 | 你的供应链有什么独特能力？（自有工厂 = 重要优势） | 核心能力清单 |
| 4.2 | 你能保证的品质底线是什么？（确定性交付） | 品控标准 |
| 4.3 | 售后/客服体验如何设计？（终值体验） | 售后方案 |
| 4.4 | 库存和交期能稳定在什么水平？ | 供应稳定性 |
| 4.5 | 哪些能力是竞品无法快速复制的？ | 能力护城河 |

**输出格式**：
```
核心能力：
1. [能力1]：[如何支撑战略存在层的承诺]
2. [能力2]：[如何支撑战略存在层的承诺]

确定性承诺：[用户每次购买都能确定得到什么]
能力护城河：[竞品需要多久才能追上？]
```

**一致性检查 ↓**：
- 能力圈层的每个能力是否服务于战略存在层的使命？
- 如果使命是"高端衣物护理"，能力是否包含精密品控？

### 第3层：资源结构层设计

**核心问题：用什么功能、材料、工艺来实现承诺？**

| # | 问题 | 输出 |
|---|------|------|
| 3.1 | 基于竞品差评，哪些功能是 Must-have？ | 必备功能清单 |
| 3.2 | 基于战略存在层的使命，需要什么材料/工艺来匹配？ | 材料工艺选择 |
| 3.3 | 有什么"兴奋型"功能能创造意外惊喜？ | A型功能清单 |
| 3.4 | 参数规格如何定义？（尺寸、重量、功率等） | 核心参数表 |
| 3.5 | 哪些功能应该 Cut？（不增加确定性的功能） | Cut 清单 |

**输出格式**：
```
MVP 功能矩阵：
- Must-have: [功能1], [功能2], ...（解决差评痛点 + 确定性交付）
- Nice-to-have: [功能3], [功能4], ...（提升体验但非必须）
- Cut: [功能5], [功能6], ...（不增加确定性，增加成本/复杂度）

材料选择：[材料] — 选择理由：[如何匹配品牌使命]
核心参数：[参数表]
```

**一致性检查 ↓**：
- 每个功能是否能被能力圈层持续交付？
- 材料选择是否匹配战略存在层的定位？（高端使命 + 廉价材料 = 矛盾）

### 第2层：角色框架层设计

**核心问题：你的产品把用户定义为什么人？**

| # | 问题 | 输出 |
|---|------|------|
| 2.1 | 目标用户是谁？（不是"25-35岁女性"，是"追求精致生活的都市租房族"） | 场景化画像 |
| 2.2 | 用户使用你的产品时，TA 希望自己是什么形象？ | 用户角色 |
| 2.3 | 这个角色是触达用户的真实自我还是社会角色？ | 自我/角色判断 |
| 2.4 | Listing 标题应该用什么人群/场景词？ | 标题关键词 |
| 2.5 | 五点描述的语气应该是什么调性？ | 文案调性 |

**输出格式**：
```
目标用户：[场景化描述，不是人口统计学]
用户角色："我是一个[角色]，我需要[场景化需求]"
自我/角色：[触达真实自我 / 满足社会角色] — [依据]
Listing 定位方向：
  - 标题关键词：[场景词] + [人群词] + [核心功能词]
  - 五点调性：[专业/温暖/简洁/...] — 因为目标用户是[...]
```

**一致性检查 ↓**：
- 角色定位是否与资源结构层的功能匹配？（定位"专业人士"但功能是入门级 = 矛盾）
- 角色是否与战略存在层的使命一致？

### 第1层（顶层）：感知层设计

**核心问题：用户第一眼看到什么？记住什么？**

| # | 问题 | 输出 |
|---|------|------|
| 1.1 | 主图应该传递什么核心信息？（一眼 get 价值） | 主图策略 |
| 1.2 | 产品外观的核心差异化在哪？ | 外观设计方向 |
| 1.3 | 包装开箱体验如何设计？（峰值体验） | 开箱设计 |
| 1.4 | 首次使用体验如何设计？（峰值体验） | 首次使用设计 |
| 1.5 | 颜色/材质/形态给人什么感觉？ | 感官设计 |

**输出格式**：
```
主图策略：[核心信息] + [差异化视觉元素]
外观差异化：[具体方向] — 因为[战略存在层使命要求/角色框架定位要求]
峰值体验设计：
  - 开箱瞬间：[设计要点]
  - 首次使用：[设计要点]
终值体验设计：
  - 最后一次交互：[设计要点]（决定Review内容）
```

**一致性检查 ↓**：
- 感知层是否忠实表达了战略存在层的使命？
- 外观是否匹配角色框架层的用户期望？
- 主图是否让目标用户一眼就知道"这是给我的"？

## 四、五层一致性总检

设计完成后，执行总检：

| 检查项 | 方法 | 通过标准 |
|--------|------|---------|
| 战略↔能力 | 使命是否有能力支撑？ | 每条使命都有对应能力 |
| 能力↔资源 | 能力能否持续交付这些功能？ | 无"承诺了但做不到"的功能 |
| 资源↔角色 | 功能是否匹配目标用户期望？ | 无"定位高端但功能低端"矛盾 |
| 角色↔感知 | 外观是否让目标用户认出"给我的"？ | 用户画像与视觉语言一致 |
| 战略↔感知 | 第一印象是否传递品牌使命？ | 看主图就知道品牌代表什么 |

**矛盾处理**：
- 发现矛盾时，**从底层修改**（改战略存在层/能力圈层），不要从顶层修（不要为了好看而改使命）
- 除非底层过度理想化不切实际，那就调整底层使其可执行

## 五、峰终定律应用

在设计模式中嵌入峰终定律：

### 峰值体验设计（2 选 1 集中资源）

| 候选峰值 | 设计要点 | 适用品类 |
|---------|---------|---------|
| 开箱瞬间 | 包装设计、首次亮相的仪式感 | 礼品类、高客单价、外观驱动 |
| 首次使用 | 即插即用、零学习成本、立即见效 | 功能型、工具类、日用品 |
| 效果呈现 | 使用后的可见效果（干净/整齐/光亮） | 清洁类、美容类、收纳类 |

### 终值体验设计

| 终值场景 | 设计要点 | 影响 |
|---------|---------|------|
| 售后客服 | 响应速度、解决态度、超出预期 | 直接决定 Review 内容 |
| 产品寿命终点 | 耐用性、可替换部件 | 决定复购和口碑 |
| 最后一次使用 | 产品报废时的体验 | 影响品牌印象 |

## 七、与诊断模式的协作

| 场景 | 做法 |
|------|------|
| 先诊断再设计 | 用诊断模式分析竞品最弱层 → 在设计模式中针对性强化这一层 |
| 先设计再诊断 | 产品上架后用诊断模式验证五层是否如设计预期 |
| 迭代循环 | 诊断→找病根→设计修复方案→实施→再诊断 = 持续进化 |

## 八、注意事项

1. **从底层开始设计**：战略存在层是地基。地基歪了，上面再好看也没用
2. **每层都要回答"为什么"**：不是"这个功能好"，而是"这个功能为什么服务于我的使命"
3. **一致性 > 完美性**：五层一致但每层80分 > 某层100分但层间矛盾
4. **用户的真实自我**：产品触达用户的真实自我（自我），不是社会角色。除非你做的是B2B
5. **恐惧是最好的战略存在层**：如果你的产品能击中恐惧，战略存在层自动稳固
6. **工厂是能力圈层的王牌**：主人有自有工厂 = 能力圈层天然优势，在设计时充分利用这一点
