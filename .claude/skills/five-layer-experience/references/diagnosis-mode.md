# 诊断模式方法论

> 从用户可见的问题出发，逐层深挖到根因。方向：感知层 → 战略存在层（从上到下）

## 一、数据采集与信号映射

### 必须采集的 Sorftime 数据

| 数据源 | 工具 | 提取信号 |
|--------|------|---------|
| 产品详情 | `product_detail` | 标题、价格、星级、评论数、品牌、卖家、上架时间、大类排名 |
| 负面评论 | `product_reviews`（Negative） | 各类差评痛点分类与频次 |
| 正面评论 | `product_reviews`（Positive） | 用户认可的价值点 |
| 产品报告 | `product_report` | 月销量、月销额、趋势 |
| 关键词曝光 | `competitor_product_keywords` | 自然排名的关键词数量和位置 |
| 产品趋势 | `product_trend` | 销量/排名趋势（上升/下降/震荡） |

### 降级策略

如果 Sorftime MCP 不可用：
- 要求用户提供 Seller Central 数据截图或文字
- 使用 WebSearch 搜索 Amazon 产品页面信息
- 标注 ⚠️ 数据来源为非MCP

## 二、五层评分标准

### 第1层：感知层（Perception）— 第一印象

**评分维度**：用户第一眼看到产品时的感受

| 分数 | 标准 | 数据信号 |
|------|------|---------|
| 5 | 主图一眼传递价值差异，包装/外观有高级感 | 主图点击率高（推算：高排名+高转化）；正面Review提到"好看""高级""质感" |
| 4 | 主图清晰专业，外观不丑但无特色 | 在搜索结果中不突出但不拖后腿 |
| 3 | 主图及格，外观普通 | 与竞品主图无明显区别 |
| 2 | 主图质量差或误导，外观廉价 | 差评提到"实物与图片不符""看起来廉价" |
| 1 | 主图劝退，外观引发负面情绪 | 大量差评提到外观/包装问题 |

**数据信号提取**：
- 正面评论中搜索关键词：`looks`、`design`、`beautiful`、`sleek`、`premium`、`quality look`
- 负面评论中搜索关键词：`cheap`、`ugly`、`flimsy`、`looks different`、`disappointed`
- 两类出现频率的对比 = 感知层评分依据

### 第2层：角色框架层（Role Frame）— 目标用户定位

**评分维度**：产品是否清晰定义了"为谁而做"

| 分数 | 标准 | 数据信号 |
|------|------|---------|
| 5 | 标题/五点精准定位细分人群，Listing 说的就是"你" | 标题包含明确场景/人群词；Review中用户自我认同"as a [role]..." |
| 4 | 定位可见但不够锐利，覆盖面偏广 | 标题通用，但A+页面有场景图 |
| 3 | 谁都能用 = 谁都不会特别想要 | 标题纯参数堆砌，无人群/场景词 |
| 2 | 定位混乱，标题说A但产品是B | Review中出现"不是我预期的""买错了" |
| 1 | 完全没有定位，不知道卖给谁 | 大量"这是什么？""有什么用？"类评论 |

**数据信号提取**：
- 标题分析：是否包含场景词（`for home`、`for travel`、`for kids`）或人群词（`professional`、`beginner`）
- Review中搜索：`as a mom`、`as a chef`、`for my apartment`、`perfect for` — 匹配度高=角色框架清晰
- 关键词覆盖：`competitor_product_keywords` 中场景/人群类关键词的占比

### 第3层：资源结构层（Resources）— 功能/材料/工艺

**评分维度**：产品用什么满足用户需求

| 分数 | 标准 | 数据信号 |
|------|------|---------|
| 5 | 功能完整+材料优质+工艺精良，无明显短板 | 星级≥4.5；差评中功能/材料类投诉<10% |
| 4 | 核心功能好，个别非核心有瑕疵 | 星级4.0-4.4；差评集中在1-2个非核心功能 |
| 3 | 功能基本达标但无亮点，材料中规中矩 | 星级3.5-4.0；"还行""一般""凑合用" |
| 2 | 核心功能有缺陷，或材料明显低质 | 星级3.0-3.5；差评集中在核心功能 |
| 1 | 产品基本不能用 | 星级<3.0；"不工作""坏了""退货" |

**数据信号提取**：
- 负面Review分类：功能缺陷 / 材料问题 / 工艺粗糙 / 尺寸不合 — 各占比多少？
- 正面Review分类：哪些功能被点名表扬？哪些材料被认可？
- 与竞品对比：同价位产品的功能覆盖度对比

### 第4层：能力圈层（Capabilities）— 持续交付能力

**评分维度**：卖家能否持续稳定地提供一致的体验

| 分数 | 标准 | 数据信号 |
|------|------|---------|
| 5 | 品质稳定，客服响应好，库存充足 | Review时间线品质一致；无"以前好现在差"；卖家回复率高 |
| 4 | 基本稳定，偶有波动 | 少量"第二次买不如第一次"类评论 |
| 3 | 品质有波动，售后一般 | 出现"质量下降""批次不一致"评论 |
| 2 | 品质明显下滑或不稳定 | 近3个月差评率明显上升；"以前4星现在只值2星" |
| 1 | 品控崩溃，客服失联 | "完全不同的产品""卖家不理人""骗子" |

**数据信号提取**：
- Review时间分析：最近3个月 vs 6个月前的差评率变化
- `product_trend`（Rank）：BSR趋势是否稳定
- 负面Review中搜索：`used to`、`before`、`changed`、`worse`、`not the same`
- 客服相关：`customer service`、`no response`、`refund`

### 第5层：战略存在层（Strategic Why）— 为什么存在

**评分维度**：品牌是否有清晰的存在理由和价值主张

| 分数 | 标准 | 数据信号 |
|------|------|---------|
| 5 | 有品牌使命+用户认同品牌价值+形成品牌忠诚 | 品牌关键词搜索量高；Review提到品牌名称表达信任；"这个品牌的我都买" |
| 4 | 有品牌故事，用户有印象但忠诚度一般 | 有品牌旗舰店；部分Review提到品牌 |
| 3 | 有品牌注册但无品牌建设 | 品牌旗舰店空壳；无品牌搜索量 |
| 2 | 纯白牌，完全靠价格和功能竞争 | 无品牌旗舰店；用户Review从不提品牌名 |
| 1 | 品牌负面认知 | "这个品牌不要买""假货""山寨" |

**数据信号提取**：
- `keyword_detail`（品牌名）：品牌词搜索量
- 品牌名在Review中被提及的频率和情感倾向
- 品牌旗舰店是否存在且有内容
- 产品线广度：同品牌其他产品的表现

## 三、病根定位逻辑

### 3.1 找最弱层

五层评分完成后，找到得分最低的层 = 病根层。

### 3.2 层间传导分析

底层问题会向上传导：

```
战略存在层有问题 → 角色框架层模糊 → 资源结构层乱配 → 感知层无差异
能力圈层有问题 → 资源结构层不稳定 → 差评增加 → 感知层恶化
```

**判定规则**：
- 如果底层（4-5层）得分低且上层也低 → 病根在底层，上层是症状
- 如果底层得分高但上层低 → 上层是独立问题（例如产品好但Listing差）
- 如果只有中间层低 → 针对性修复即可

### 3.3 "改错了层"检测

常见误诊：
| 症状 | 错误处方 | 正确处方 |
|------|---------|---------|
| 转化率低 | 换主图（感知层） | 检查角色框架层是否定位模糊 |
| 差评多 | 增加功能（资源结构层） | 检查是否是能力圈层（品控）问题 |
| 销量下滑 | 降价（资源结构层） | 检查战略存在层是否被新品类颠覆 |
| Review差 | 投诉客服（能力圈层） | 检查资源结构层产品是否有设计缺陷 |

## 五、注意事项

1. **每个评分必须有数据依据**，不能凭直觉打分
2. **底层问题优先修**，不要只治症状不治病根
3. **资源有限时用峰终定律**：集中资源做好1-2个峰值体验，不要平均用力
4. **区分品类限制和产品问题**：整个品类都3星不是个别产品的问题
5. **关注层间矛盾**：如果战略存在层定位"高端"但感知层看起来"廉价"，这个矛盾比任何单层问题都严重
