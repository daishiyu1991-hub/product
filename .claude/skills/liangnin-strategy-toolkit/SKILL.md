# 梁宁战略工具箱

> 5 个梁宁理论工具，一个 Skill 统一调用。
> 与 `five-layer-experience` Skill（五层体验双向工具）互补，不重叠。

## 工具总览

| 模式 | 命令 | 解决什么问题 | 理论来源 |
|------|------|------------|---------|
| 🚀 三级火箭 | `rocket` | 一个品不是品牌 → 怎么从1个产品规划出产品矩阵？ | 创新模式·三级火箭 |
| 🔄 飞轮诊断 | `flywheel` | 产品的增长飞轮转起来了吗？卡在哪？刹车在哪？ | 增长思维·增强回路/调节回路 |
| 🌐 价值网分析 | `valuenet` | 你是在别人的地盘打仗（必输），还是开辟了新战场？ | 创新模式·颠覆式创新/价值网 |
| 🎬 场景拆解 | `scene` | 你的产品在什么场景下被需要？场景不清 = 没有产品 | 机会判断·场景思维 |
| ✅ 确定性审计 | `certainty` | 用户依赖你是因为你持续提供确定性 — 你的确定性稳吗？ | 系统能力·确定性 |

## 使用方法

```
/liangnin-strategy-toolkit rocket "衣物护理"
/liangnin-strategy-toolkit flywheel B0F5PZPHTC US
/liangnin-strategy-toolkit valuenet "吸烫机3800Pa"
/liangnin-strategy-toolkit scene "吸烫机" US
/liangnin-strategy-toolkit certainty B07HF3X6Y4 US
```

不带参数 → 展示工具菜单让用户选择

## 模式判断

解析第一个参数：
- `rocket` / `火箭` / `矩阵` / `产品线` → 三级火箭模式
- `flywheel` / `飞轮` / `回路` / `增长` → 飞轮诊断模式
- `valuenet` / `价值网` / `颠覆` / `创新` → 价值网分析模式
- `scene` / `场景` / `情绪` → 场景拆解模式
- `certainty` / `确定性` / `审计` / `稳定` → 确定性审计模式
- ASIN格式（B0开头10位）→ 自动判断：如果有第二个参数是模式词就用该模式，否则默认飞轮诊断
- 其他 → 展示菜单

## 执行流程

**先读取本文件 → 判断模式 → 读取 `references/` 下的对应方法论文件 → 执行**

### 🚀 三级火箭模式（rocket）

读取 `references/three-stage-rocket.md`

**输入**：品类名称 / 现有产品ASIN / 品牌名
**流程**：
1. 了解用户现有产品线（主动询问）
2. 分析品类生态和用户动线
3. 设计三级火箭：
   - 第一级（头部流量）：用什么产品获客？
   - 第二级（沉淀用户）：怎么让用户留在你的品牌？
   - 第三级（商业闭环）：利润从哪来？
4. 结合工厂能力评估可行性（注意：用户不一定在此品类有工厂）
5. 输出产品矩阵规划

**典型场景**：
- "我吸烫机卖起来了，下一个产品做什么？"
- "我的灯具产品线怎么规划？"
- "怎么从一个爆品变成一个品牌？"

### 🔄 飞轮诊断模式（flywheel）

读取 `references/flywheel.md`

**输入**：ASIN + 站点（已上架产品）或产品概念（规划中产品）
**流程**：
1. 已上架产品：用 Sorftime 拉取产品数据（销量趋势、Review趋势、关键词排名趋势）
2. 画出该产品的增强回路（正向飞轮）
3. 诊断飞轮每个节点是否在正常运转
4. 识别调节回路（刹车/风险点）
5. 给出加速飞轮 / 修复断裂环节的具体行动

**典型场景**：
- "我的产品销量为什么起不来？"
- "Review越来越差怎么回事？"
- "飞轮感觉卡住了"

### 🌐 价值网分析模式（valuenet）

读取 `references/value-network.md`

**输入**：产品概念 + 品类名称
**流程**：
1. 识别当前品类的**主流价值网**（谁是玩家、靠什么竞争、用户看重什么）
2. 判断你的产品在哪个价值网：
   - 同一张网 → 正面竞争（慎重）
   - 低端颠覆 → 从大玩家看不上的细分切入
   - 新市场颠覆 → 创造全新使用场景/价值网
3. 评估颠覆可行性
4. 给出竞争策略建议

**典型场景**：
- "吸烫机和传统蒸烫机是同一个市场吗？"
- "我的产品会被大品牌碾压吗？"
- "这个品类适合低端切入还是高端切入？"

### 🎬 场景拆解模式（scene）

读取 `references/scene-decomposer.md`

**输入**：产品名称 / ASIN + 站点
**流程**：
1. 用 Sorftime 拉取关键词数据和 Review 数据
2. 从关键词和 Review 中提取场景信号
3. 拆解每个场景为 **时间 × 空间 × 情绪**
4. 按场景规模（搜索量）排序
5. 识别"未被占领的场景"
6. 输出场景化 Listing 方向

**典型场景**：
- "这个产品应该主打什么场景？"
- "Listing五点怎么写才有画面感？"
- "有没有我没看到的使用场景？"

### ✅ 确定性审计模式（certainty）

读取 `references/certainty-audit.md`

**输入**：ASIN + 站点
**流程**：
1. 用 Sorftime 拉取产品数据（趋势、Review、排名）
2. 审计 4 个维度的确定性：
   - 品质确定性：品质是否稳定？
   - 交付确定性：库存/物流是否可靠？
   - 体验确定性：用户每次使用是否一致？
   - 价值确定性：价格/价值比是否稳定？
3. 找出确定性最弱的环节
4. 给出修复建议

**典型场景**：
- "为什么我的老品最近销量下滑？"
- "如何判断一个产品能否长期卖？"
- "竞品的确定性有没有漏洞可以利用？"

## 跨工具组合用法

| 组合 | 场景 | 顺序 |
|------|------|------|
| 价值网 → 场景 | 先确认你不在别人的价值网 → 再找你的专属场景 | 串行 |
| 飞轮 → 确定性 | 飞轮卡住了 → 审计哪个确定性断了 | 串行 |
| 场景 → 三级火箭 | 找到核心场景 → 围绕场景规划产品矩阵 | 串行 |
| 价值网 + 飞轮 | 新品上市前：价值网判断能不能赢 + 飞轮设计怎么赢 | 并行 |

## 与 Phase 流程的关系

> **本工具箱的 5 个工具已整合到精益开发流程中，但仍可独立调用。**

| 本Skill工具 | 整合到的 Phase | 独立调用场景 |
|------------|--------------|------------|
| 场景拆解 | **Phase 3** MVP 蓝图 | 脱离选品流程，单独分析某品类的场景机会 |
| 价值网 | **Phase 3** MVP 蓝图 | 单独判断某个产品是否在别人的价值网竞争 |
| 三级火箭 | **Phase 4** 战略验证 | 已有产品线，单独规划产品矩阵 |
| 飞轮诊断 | **Phase 4** 战略验证 | 已上架产品，单独诊断飞轮卡点 |
| 确定性审计 | **Phase 4** 战略验证 | 已上架产品，单独审计确定性 |

| 本Skill工具 | 互补的现有工具 | 关系 |
|------------|--------------|------|
| 三级火箭 | Phase 9（规模化决策）| 三级火箭规划产品矩阵 → Phase 9 逐个评估是否值得投入 |
| 飞轮诊断 | Phase 7-8（复盘+迭代）| Phase 7 发现问题 → 飞轮诊断定位断裂环节 → Phase 8 修复 |
| 价值网 | Phase 1（选品分析）| Phase 1 看数据 → 价值网看格局 → 一起决定做不做 |
| 场景拆解 | 文案操盘手Agent | 场景拆解输出 → 文案操盘手用场景写Listing |
| 确定性审计 | 运营分析师Agent | 确定性审计 → 运营分析师执行修复方案 |
