# 三级火箭方法论

> 来源：梁宁《产品思维30讲》创新模式 + 《增长思维30讲》模式模块
> 核心：一个产品不是品牌。三级火箭 = 从一个产品规划出完整产品矩阵的方法。

## 原理

```
第一级（头部流量）→ 第二级（沉淀用户）→ 第三级（商业闭环/利润）
```

- **第一级**：用高频/低门槛产品获取大量用户。不赚钱甚至亏钱都行，目的是流量
- **第二级**：把流量用户沉淀到你的品牌（关注品牌店铺、加购、收藏、复购）
- **第三级**：用高毛利产品/服务实现商业闭环

### 亚马逊电商版三级火箭

| 级 | 传统互联网 | 亚马逊电商版 | 核心指标 |
|----|----------|------------|---------|
| 第一级 | 免费产品/爆款引流 | **引流品**：低价高销量，占领搜索心智 | 月销量、搜索排名 |
| 第二级 | 用户池/社群/会员 | **关联品**：通过"经常一起购买"+"品牌店铺"沉淀用户 | 关联购买率、品牌店铺访问 |
| 第三级 | 高毛利服务/增值 | **利润品**：高客单价、高毛利、品牌溢价产品 | 毛利率、LTV |

## 执行步骤

### Step 1: 现状盘点

询问用户：
- 你现在有几个产品/品类？
- 哪个是主力产品？月销多少？
- 你的工厂能力覆盖哪些品类？（重要：用户可能只在某个品类有工厂）
- 你的品牌注册在哪些品类？

### Step 2: 识别第一级（引流品）

**选择标准**：
| 标准 | 说明 | 量化信号 |
|------|------|---------|
| 高搜索量 | 品类核心词月搜索量大 | `keyword_detail` 月搜索量 >50K |
| 高购买频次 | 用户可能反复购买 | 看品类 Review 中"again""repurchase"频率 |
| 低价格带 | 用户低门槛尝试 | 品类均价 <$30 |
| 低竞争壁垒 | 不需要大量资金/专利 | Top3 品牌集中度 <40% |

**输出**：第一级候选产品清单 + 每个的流量获取潜力评估

### Step 3: 设计第二级（沉淀品/关联品）

**核心问题**：买了第一级的用户，还需要什么？

**关联路径设计**：
```
用户买了 [引流品] → 自然会需要 → [关联品]
```

| 关联类型 | 说明 | 亚马逊实现方式 |
|---------|------|--------------|
| **配件型** | 引流品的配件/耗材 | FBT（经常一起购买）|
| **升级型** | 引流品的高端版本 | 品牌店铺内导流 |
| **场景型** | 同一场景下的其他产品 | 品牌旗舰店场景页 |
| **互补型** | 功能互补的不同品类 | A+ 交叉推荐 |

**输出**：第二级候选产品 + 与第一级的关联路径

### Step 4: 规划第三级（利润品）

**选择标准**：
| 标准 | 说明 |
|------|------|
| 高毛利 | 毛利率 >40%，有品牌溢价空间 |
| 品牌依赖 | 用户需要信任品牌才会买（不是冲动消费）|
| 差异化壁垒 | 有专利/设计/技术门槛 |
| 高客单价 | 单价 >$50 |

**利润品常见形态**：
- 套装/组合包（把多个关联品打包，提高客单价）
- 高端版/Pro版（同品类的高配版本）
- 订阅型耗材（滤芯、替换头等）
- 定制化产品（刻字、选色等）

### Step 5: 火箭可行性评估

| 检查项 | 通过标准 |
|--------|---------|
| 第一级 → 第二级路径是否自然？ | 用户不需要被说服就会考虑买关联品 |
| 第二级 → 第三级路径是否成立？ | 用户对品牌建立信任后愿意买高价产品 |
| 三级之间是否共享供应链？ | 至少2级能用同一产线/供应商 |
| 三级之间是否共享流量？ | 关键词/用户群有重叠 |
| 整体品牌故事是否一致？ | 三级产品能用一个品牌故事串联 |

### Step 6: 输出产品矩阵图

```
┌─────────────────────────────────────────────┐
│              [品牌名] 产品矩阵              │
├──────────┬──────────────┬───────────────────┤
│  第一级   │    第二级     │      第三级       │
│  引流品   │   关联品      │     利润品        │
├──────────┼──────────────┼───────────────────┤
│ [产品A]  │  [产品B]     │    [产品D]        │
│ $XX      │  $XX         │    $XXX           │
│ 目标:流量 │  目标:沉淀    │    目标:利润       │
│          │  [产品C]     │    [产品E]        │
│          │  $XX         │    $XXX           │
├──────────┴──────────────┴───────────────────┤
│ 关联路径：A → B(配件) → D(升级)              │
│          A → C(场景) → E(套装)              │
└─────────────────────────────────────────────┘
```

## 常见反模式

| 反模式 | 问题 | 修正 |
|--------|------|------|
| "先做利润品" | 没有流量基础，利润品卖不动 | 先用低价品获客 |
| "三级产品毫无关联" | 无法形成用户动线 | 三级必须有自然关联 |
| "第一级就想赚钱" | 引流品加价 → 流量减少 → 后续断链 | 第一级允许低利润甚至微亏 |
| "只做第一级" | 永远在价格战中内卷 | 必须规划上升路径 |

## 注意事项

1. **用户不一定在每个品类都有工厂** — 询问清楚制造能力边界
2. **三级火箭不是一步到位** — 先做好第一级，验证后再推第二三级
3. **品类跨度不宜过大** — 灯具和吸烫机不适合放在同一个三级火箭里
4. **亚马逊的关联推荐算法是关键** — FBT、品牌店铺、A+ 交叉引流都是沉淀手段
