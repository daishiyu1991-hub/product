# 场景拆解方法论

> 来源：梁宁《产品思维30讲》机会判断 — 场景思维
> 核心：场景不清 = 没有产品。产品不是功能的集合，是场景的解决方案。

## 原理

### 什么是场景？

场景 = **时间 + 空间 + 情绪**

- **时间**：什么时候会需要这个产品？（早上赶着上班 / 周末在家放松 / 出差途中）
- **空间**：在什么环境下使用？（小公寓 / 办公室 / 酒店 / 户外）
- **情绪**：用户当时的情绪状态？（焦虑赶时间 / 放松享受 / 沮丧解决问题）

### 场景 vs 功能

| | 功能思维 | 场景思维 |
|--|---------|---------|
| 思考方式 | "我的产品有XX功能" | "用户在XX场景下需要我" |
| Listing 写法 | "3800Pa大吸力" | "30秒一件，早上多睡10分钟" |
| 竞争方式 | 参数比拼 | 场景占领 |
| 用户感知 | "功能挺多" | "这就是为我设计的" |

### 为什么场景比功能重要？

1. **用户不买功能，买场景解决方案** — 没人买"3800Pa吸力"，买的是"衣服一次过就平整"
2. **场景定义竞争边界** — 同一个产品，换一个场景就是不同的竞品
3. **场景决定定价权** — 同样的产品，"解决赶时间焦虑"比"熨衣服工具"定价更高
4. **场景指导 Listing** — 场景清晰 = 文案有画面感 = 转化率高

## 执行步骤

### Step 1: 数据采集 — 从关键词和 Review 中提取场景信号

**关键词信号**（场景入口）：
- `keyword_extends` → 核心关键词的延伸词，找场景化长尾词
- `keyword_search_results` → 场景词搜索出来的产品，看市场是否存在
- `keyword_detail` → 场景词的搜索量，评估场景大小
- `keyword_trend` → 场景词的趋势，判断场景是否在增长

**关键词场景信号提取规则**：
| 关键词模式 | 场景信号 | 示例 |
|-----------|---------|------|
| [产品] + for + [场景] | 使用场景 | "steamer for travel" → 旅行场景 |
| [产品] + for + [人群] | 用户画像 | "steamer for men" → 男性用户 |
| [场景形容词] + [产品] | 场景约束 | "portable steamer" → 移动场景 |
| [产品] + [空间词] | 使用空间 | "garment steamer closet" → 衣柜场景 |
| [动作] + [产品] | 使用方式 | "handheld steamer" → 手持场景 |

**Review信号**（场景验证）：
- `product_reviews`（Top 3 产品，Positive + Negative）
- 提取 Review 中的场景描述

**Review 场景信号提取规则**：
| Review 关键词 | 场景信号 | 权重 |
|-------------|---------|------|
| "I use it when..." / "I use it for..." | 直接场景描述 | 高 |
| "every morning" / "before work" | 时间信号 | 高 |
| "in my apartment" / "small space" | 空间信号 | 高 |
| "so frustrated" / "love that" | 情绪信号 | 中 |
| "my husband" / "my kids" | 用户画像 | 中 |
| "instead of ironing board" / "replaced my iron" | 替代场景 | 高 |
| "travel" / "business trip" / "hotel" | 移动场景 | 高 |

### Step 2: 场景清单 — 拆解为 时间 × 空间 × 情绪

将采集到的信号组装为完整场景：

```
场景格式：
[场景名称]
├── 时间: [什么时候]
├── 空间: [什么环境]
├── 情绪: [什么心理状态]
├── 触发动作: [什么事件触发了需求]
├── 用户画像: [什么人]
├── 搜索入口: [用什么关键词找到你]
└── 搜索量: [关键词月搜索量]
```

**示例 — 吸烫机场景拆解**：
```
场景1: 早上出门前快速整理
├── 时间: 工作日早上 7:00-8:00
├── 空间: 小公寓/卧室
├── 情绪: 焦虑、赶时间
├── 触发动作: 发现今天要穿的衣服有褶皱
├── 用户画像: 上班族，城市公寓，没有熨衣板空间
├── 搜索入口: "quick steamer" "portable garment steamer"
└── 搜索量: XX,XXX

场景2: 出差/旅行途中
├── 时间: 出差前一晚/到达酒店后
├── 空间: 酒店房间
├── 情绪: 需要体面，不想邋遢
├── 触发动作: 衣服在行李箱里被压皱
├── 用户画像: 商务人士，频繁出差
├── 搜索入口: "travel steamer" "portable steamer for clothes"
└── 搜索量: XX,XXX
```

### Step 3: 场景排序 — 按规模和匹配度排序

| 排序维度 | 权重 | 量化方法 |
|---------|------|---------|
| **场景规模** | 40% | 对应关键词月搜索量总和 |
| **场景-产品匹配度** | 30% | 你的产品功能在这个场景中的完成度（0-100%）|
| **场景竞争度** | 20% | 这个场景词下的竞品数量和质量 |
| **场景增长性** | 10% | 场景词搜索量是否在增长 |

**排序输出**：
```
场景排名：
1. [场景名] — 综合分 XX — 🟢 主攻
2. [场景名] — 综合分 XX — 🟡 辅攻
3. [场景名] — 综合分 XX — 🟡 辅攻
4. [场景名] — 综合分 XX — ⚪ 覆盖但不主打
5. [场景名] — 综合分 XX — ❌ 放弃（不匹配）
```

### Step 4: 未被占领的场景识别

**"空白场景"发现方法**：

1. **关键词有搜索量但无专门产品**
   - 方法：`keyword_search_results` 搜场景词 → 看结果中有无专门针对该场景的产品
   - 信号：搜索结果全是通用产品，没有场景化产品 → 机会

2. **Review 中频繁提及但无人满足**
   - 方法：统计 Top 产品 Review 中场景描述 → 看是否有被忽略的高频场景
   - 信号：>10% 的 Review 提到某场景但现有产品不适合 → 机会

3. **跨品类场景迁移**
   - 方法：看其他品类是否有类似场景的成功案例
   - 信号：A 品类在某场景做得好，但 B 品类（你的品类）没人做 → 机会

### Step 5: 场景化 Listing 方向

将场景拆解转化为具体的 Listing 指导：

| Listing 元素 | 场景化写法 | 非场景化写法（反例）|
|-------------|----------|------------------|
| **标题** | 嵌入主攻场景关键词 | 堆砌功能参数词 |
| **主图** | 展示场景使用画面 | 纯产品白底图 |
| **五点** | 每点对应一个场景的解决方案 | 每点列一个功能参数 |
| **A+** | 场景故事线：问题→使用→结果 | 产品结构分解图 |
| **视频** | 真实场景演示（晨间整理/出差收纳）| 产品功能演示 |

**五点场景化模板**：
```
Bullet 1: [主攻场景] — 用户的痛点画面 → 产品如何在30秒内解决
Bullet 2: [辅攻场景1] — 另一个高频使用场景的画面感描述
Bullet 3: [核心差异功能] — 支撑场景解决方案的关键技术/设计
Bullet 4: [辅攻场景2 或 广泛适用性] — 多材质/多场合适用
Bullet 5: [信任保障] — 安全认证/质保/售后
```

## 注意事项

1. **场景必须有数据支撑** — 每个场景都要有关键词搜索量或 Review 频率作为证据，不能凭感觉编场景
2. **场景≠功能** — "3800Pa吸力"不是场景，"早上出门前30秒搞定皱衣服"才是场景
3. **一个产品最多主攻 1-2 个场景** — 场景太多 = 没有场景。Listing 空间有限，必须聚焦
4. **场景决定你的竞品是谁** — 主攻"旅行场景"你的竞品是便携蒸烫机；主攻"日常家用"你的竞品是熨斗
5. **场景词是 Listing 埋词的金矿** — 场景化长尾词通常竞争低、转化高
6. **场景和五层体验的关系** — 场景是角色框架层的外在表现（不同场景 = 不同角色），可以和 `five-layer-experience` Skill 联合使用
