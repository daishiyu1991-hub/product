# MVP 产品蓝图（精益开发 Phase 3）

## 定位

`lean-product-develop-mvp-blueprint` 是精益产品开发闭环的第三阶段，衔接上游的选品分析（`zach-product-research`）和需求验证（`zach-feature-demand-validator`），将数据结论转化为**可执行的最小可行产品规格书**。

核心问题：**最少做什么就能验证市场？**

## 适用场景

- 选品报告已给出 GO/CONDITIONAL GO 结论，需要定义具体产品规格
- 需求验证已确认功能为真需求，需要定义 MVP 范围
- 团队准备首批下单前，需要明确产品定义、成本预算和测试标准
- 需要生成 Listing 上架方案（标题/五点/A+/图片/定价）

## 前置条件

- 上游选品报告或需求验证报告（推荐，非强制）
- Python 3.9+，安装 `openpyxl`
- 目标站点（默认 US）

## 使用方法

### 方式1：指定品类
```
/lean-product-develop-mvp-blueprint bluetooth speaker US
```

### 方式2：关联上游报告
```
/lean-product-develop-mvp-blueprint bluetooth speaker US --source ./工作成果/brands/_选品/市场调研/bluetooth-speaker/v1_20260320/
```

### 方式3：自然语言
```
帮我定义蓝牙音箱的 MVP 产品规格，预算 10 万
```

## 执行步骤

当调用此 Skill 时，执行以下流程：

1. **读取完整方法论** — 加载 `skills/lean-product-develop-mvp-blueprint/SKILL.md`
2. **读取上游数据** — 加载选品报告/需求验证报告，提取核心结论（交叉分析空白点、差评痛点、竞品规格、目标定价、关键词）
3. **用户画像构建** — 基于上游数据（搜索词语义、差评用户属性、竞品评价人群特征、品类使用场景），构建目标用户画像（见下方「用户画像构建方法论」）。用户画像是后续所有步骤的锚点——根因拆解要考虑用户使用场景，创新亮点要匹配用户偏好，配色要符合用户审美，Listing 文案要用用户的语言。
4. **痛点根因拆解** — 对每个差评痛点执行「原理层拆解」（见下方详细方法论），将「用户感受」分解为「物理/工程根因」，输出精确的改良方向和供应商沟通话术
5. **内部堆叠剖面图** — 在根因拆解之后、创新亮点之前，调研产品内部组件堆叠结构（见下方「内部堆叠剖面图方法论」）。通过 WebSearch 调研拆解/专利/工程资料，画出组件堆叠图，然后调用 `/allen-product-design` 生成带标注的内部剖面渲染图，将 Step 4 的改善点直接标注在剖面图上（如「此处加气液分离挡板」「密封圈换 EPDM」）。这张图的作用是：① 让团队/供应商一眼看懂改良方案在哪里 ② 验证外观设计方案是否兼容内部堆叠 ③ 约束后续工业设计不脱离可制造性
6. **创新亮点 & 外观差异化** — 基于用户画像偏好+品类趋势+竞品空白点，提出创新价值点（进攻型功能）和外观差异化方案（配色/表面处理/开箱体验）
7. **用户影响力排序** — 将差评改善（防守型）、创新亮点（进攻型）、外观差异化统一按「Listing 可感知度 × 购买决策影响力 × 使用体验影响力 × 成本效率」四维度加权评分排序（见下方评估模型），输出分档预算方案。对不可感知的内部改善，必须提出可视化包装方案。
8. **MVP 功能矩阵** — 基于影响力排序结论定义 Must-have / Nice-to-have / Cut 三级功能
9. **MVP 规格书** — 每个属性维度的具体规格值 + 决策依据 + 原理层解释
10. **工业设计概念图** — 完成规格书后，调用 `/allen-product-design` skill 生成概念渲染图。调用时必须将前面所有阶段积累的上下文（含用户画像 + 内部堆叠结构）完整传递（见下方「工业设计调用规范」），让 `allen-product-design` 拥有完整的产品理解来构建精准的 prompt。**注意：内部堆叠剖面图在 Step 5 已经完成，此处 allen-product-design 会使用堆叠数据约束外观设计的可行性，并根据目标用户年龄发散风格探索。**
11. **测试标准定义** — 量化的 Pass/Fail 标准（转化率/BSR/ACOS/退货率/评分/周期）
12. **成本与盈亏平衡** — 首批采购+模具+头程+FBA+广告预算+盈亏平衡点
13. **Listing 上架规划** — 标题/五点/A+/图片清单/定价策略（文案语气、卖点排序、场景选择必须匹配用户画像）
14. **供应商沟通指南** — 基于根因拆解+工业设计图，生成精确的技术改良需求表
15. **输出与验证** — 生成文档三件套 + 工业设计图集

|------|----------|------|
| 蓝图 (MD) | `[日期]_[站点]_[关键词]_MVP蓝图_v[n].md` | MVP 完整规格书（含根因拆解+影响力排序+供应商指南） |

## 下游对接

- MVP 蓝图中的测试标准 → `lean-product-develop-launch-review` 复盘时的对照基线
- Listing 规划 → 直接用于产品上架
- 根因拆解 + 供应商沟通指南 → 直接用于供应商谈判和打样需求
- 工业设计概念图 → 供应商发送打样参考（"这就是我们要做的产品"）+ Listing 主图拍摄方向参考 + 品牌视觉标准锚定

## 数据诚信规则

- **绝不捏造数据**：所有成本和市场数据必须来自 Sorftime MCP 或明确标注来源
- **标注数据来源**：每个数据点标注来源工具或渠道
- **区分事实与推测**：事实用数据支撑，推测必须标注「推测」
- **区分品类限制与设计缺陷**：根因拆解中必须区分哪些是品类固有物理极限（无法改善），哪些是竞品设计缺陷（可以改善）

## 用户画像构建方法论

### 为什么用户画像要在根因拆解之前？

因为用户画像决定了后续每一步的判断基准：

| 后续步骤 | 用户画像如何影响该步骤 |
|---------|---------------------|
| 根因拆解 | 同一个"蒸汽弱"痛点，专业用户觉得不可忍受(需加大功率)，轻量用户觉得够用(只需管理期望)。画像决定哪些痛点是真正的 deal-breaker。 |
| 创新亮点 | LED 灯对科技型用户有吸引力，对价格敏感用户是浪费成本。画像决定哪些创新值得投入。 |
| 外观配色 | 薄荷绿吸引年轻女性家庭用户，深灰吸引男性/专业用户。画像决定配色方向。 |
| 影响力排序 | "Listing 可感知度"的权重应该因用户购买决策模式而异——冲动购买型用户主图权重极高，研究型用户评价/参数权重更高。 |
| Listing 文案 | 语气、卖点顺序、场景选择必须用用户的语言说用户关心的事。 |
| 工业设计 | 传给 allen-product-design 的上下文需要包含目标用户的审美偏好。 |

### 数据来源

用户画像的构建不是凭想象，而是从已有数据中提取：

| 数据源 | 提取什么 | 怎么用 |
|--------|---------|--------|
| **搜索词语义** | 用户怎么搜这个产品？用什么词？ | "steam cleaner for home" → 家用场景；"steam cleaner for car" → 有车人群；"chemical free" → 健康敏感 |
| **竞品差评用户自述** | 差评中用户描述的使用场景和身份线索 | "senior citizen""on a cruise""for my kids" → 提取年龄/生活方式/使用场景 |
| **竞品好评用户自述** | 好评中用户赞美的点 → 真正在意什么 | "perfect for spot cleaning""great for grout" → 轻量日常清洁需求 |
| **竞品评价人群画像** | 如有 Sorftime 人群标签数据 | 直接使用 |
| **品类平均价位** | $20-50 vs $100-200 | 价位段暗示用户消费能力和期望值 |
| **品类搜索趋势** | 旺季/增长趋势 | 旺季月份暗示使用场景(春季大扫除/节前清洁) |

### 画像输出模板

用户画像必须按以下结构输出为蓝图的一个正式章节：

```
## 目标用户画像

### 主力用户 (Primary Persona) — 占比 ~60-70%
- 人物标签：[如：25-45岁女性家庭主妇/职业女性]
- 核心诉求：[如：日常轻量清洁，不想用化学清洁剂，有小孩和宠物]
- 购买决策模式：[如：刷 TikTok 看到 → Amazon 搜索 → 看主图+价格+评分 → 3分钟内决策]
- 审美偏好：[如：清新、家居感、马卡龙色系，偏好"好看又好用"的小家电]
- 价格敏感度：[如：$30-50 舒适区，>$60 犹豫，会比较竞品价格]
- 痛点优先级：[如：最怕漏水烫到小孩 > 清洁力够日常用就行 > 不想东西太重]
- 使用场景：[如：厨房灶台/浴室瓷砖缝/偶尔清洁沙发]
- 信息触达：[如：TikTok 清洁视频 → Amazon 搜索 → 受 Vine 评价影响]

### 次要用户 (Secondary Persona) — 占比 ~20-30%
- [同上结构，但描述另一类人群]

### 边缘用户 (Edge Case) — 占比 ~10%
- [不作为 MVP 核心目标，但不排斥]

### 画像对产品定义的决策影响
| 决策点 | 画像指导 |
|--------|---------|
| 配色选择 | 主力用户审美偏好 → [薄荷绿/珊瑚粉 vs 深灰/黑色] |
| LED 灯是否做 | 主力用户购买决策模式 → [冲动型=做(主图吸引) / 研究型=不一定] |
| 蒸汽弱是否要解决 | 主力用户核心诉求 → [轻量清洁=管理期望 / 专业清洁=必须升级功率] |
| Listing 第一卖点 | 主力用户痛点优先级 → [安全>效果>便利 的排序] |
| 定价策略 | 主力用户价格敏感度 → [起步价/锚定价/提价节奏] |
```

### 没有数据时怎么办？

如果上游报告缺少评价/搜索词数据，用以下方式推断：

1. **从品类定价推断消费层级**：$20-50 入门 / $50-100 中端 / $100+ 专业
2. **从 Amazon 品类浏览人群推断（WebSearch）**：搜索 "[product category] buyer demographics" / "[product] who buys it"
3. **从竞品 Listing 的图片/文案风格推断**：竞品对谁说话？用什么语气？场景图里是什么人？
4. **标注为推测**：推断的画像必须标注 ⚠️，并在上架后 30 天用 Seller Central 人群数据修正

## 痛点根因拆解方法论

### 为什么要做根因拆解？

差评痛点如果只停留在用户感受层（"漏水""蒸汽弱"），和供应商沟通时会变成模糊的需求（"帮我做个不漏水的"），结果工厂凭经验做，改良效果听天由命。

**拆解到原理层**后，供应商沟通变成精确的工程要求（"加热腔出口加气液分离挡板""密封圈从 NBR 换 EPDM"），工厂一听就能执行。

### 拆解流程

对每一个高频差评痛点，按以下 5 步拆解：

#### Step 1：归集差评原文

从差评数据中，逐条阅读，将描述同一现象的差评归为一组。注意：

- **表面上同一个关键词，可能是完全不同的问题。** 例如"漏水"可能是：密封圈老化、壳体开裂、冷水喷射、配件接口松动——这是 4 种完全不同的物理现象。
- 归集时以**物理现象**而非用户措辞为分组依据。

#### Step 2：画出产品工作原理

用简图画出产品的核心工作流程和关键部件，标注：

- 能量/物质流动路径（水 → 加热 → 蒸汽 → 喷嘴）
- 关键连接界面（密封面、配件接口、电气连接）
- 运动/受力部件（触发阀门、弹簧、按钮）

不需要精确工程图，但需要清晰到能定位**故障发生在哪个环节**。

如果不了解某个品类的内部结构，可以通过 WebSearch 搜索 "[product type] internal structure" / "[product type] teardown" / "[product type] engineering design" 来了解。

#### Step 3：对每个痛点做物理归因（5 Whys）

从用户描述的**现象**出发，逐层追问 Why，直到触达**物理/材料/结构**层面的根因：

```
用户说："It spits water"（喷水）
  Why? → 液态水从喷嘴喷出
  Why? → 水在加热腔内未完全汽化
  Why? → 加热腔容积小 + 无气液分离机构
  Why? → 成本设计取舍，基础款省掉了分离结构
  Why? → 这是品类低价段的普遍设计缺陷
  → 根因：缺少气液分离机构（挡板/螺旋通道/二级干燥腔）
```

#### Step 4：区分「品类限制」vs「设计缺陷」

每个根因必须标注是以下哪种：

| 类型 | 定义 | MVP 能否解决 | 应对策略 |
|------|------|-------------|---------|
| **设计缺陷** | 竞品偷工减料或设计失误导致的问题 | ✅ 可以 | 定向改良，纳入 MVP 规格 |
| **品类限制** | 受物理定律、电气规格、成本约束等决定的上限 | ❌ 不能 | Listing 中管理期望，不做虚假承诺 |
| **使用问题** | 用户操作不当导致的问题 | ⚠️ 部分 | 改善说明书/产品引导设计 |

**这很关键。** 如果把品类限制当成设计缺陷去改，会浪费成本且改不了；如果把设计缺陷当成品类限制放弃，会错失核心差异化机会。

#### Step 5：输出工程解决方案 + 供应商话术

对每个**可改善的设计缺陷**：

1. 列出 2-4 种工程解决方案
2. 评估每个方案的难度（⭐⭐⭐⭐⭐）和增量成本
3. 推荐 MVP 最优组合
4. 写出**供应商沟通话术**——精确到材料名称、结构名称、量化指标

### 输出格式

根因拆解报告按以下结构输出：

```
## 痛点 X — [用户感知名称]

### 用户原文（代表性差评引用）
> "xxx" / "yyy"

### 物理现象分类
这个痛点实际上包含 N 种不同物理现象：
- XA: [现象A名称]
- XB: [现象B名称]

### XA: [现象A名称]

**5 Whys 归因：**
- 现象 → Why1 → Why2 → **根因**
- 类型：设计缺陷 / 品类限制 / 使用问题

**工程解决方案：**
| 方案 | 原理 | 难度 | 增量成本 |
|------|------|------|---------|

**MVP 推荐方案：** ...

**供应商沟通话术：**
- ❌ 不要说："我要产品不XXX"
- ✅ 正确说法："[具体工程改良要求，精确到材料/结构/指标]"
```

### 根因拆解的常见品类参考

以下是一些常见品类的典型根因拆解维度，供参考：

| 品类 | 典型痛点 | 常见根因方向 |
|------|---------|-------------|
| 电器类（蒸汽/加热） | 漏水、过热、功率不足 | 密封材质与耐温等级、热管理设计、电路保护 |
| 塑料外壳类 | 开裂、变色、异味 | 材质等级（回收料/新料/增强料）、壁厚与加强筋、注塑工艺 |
| 可拆卸配件类 | 松动、难装、断裂 | 连接方式（推入/卡扣/卡口/螺纹）、公差控制、材质选择 |
| 电池类 | 续航差、充电慢、鼓包 | 电芯容量vs标称、BMS设计、充电IC规格 |
| 电机类 | 噪音大、振动、寿命短 | 电机类型（有刷/无刷）、轴承选型、减震设计 |
| 织物/软材类 | 起球、褪色、缩水 | 纤维成分、织法密度、染色工艺、预缩处理 |
| 金属件类 | 生锈、断裂、锋利毛刺 | 材质（碳钢/不锈钢/铝合金）、表面处理、冲压/去毛刺工艺 |

### 供应商沟通的核心原则

1. **用工程语言，不用用户语言。** 工厂不理解"用户觉得蒸汽弱"，但理解"加热腔增加翅片面积提升热交换效率"。
2. **给出量化指标。** 不说"密封要好"，而说"密封圈材质需 EPDM 或硅胶，硬度 50±5 Shore A，耐温 ≥200°C，压缩永久变形率 ≤20%"。
3. **区分必须做 vs 希望做。** 把改良项按 P0/P1/P2 排优先级，让工厂知道哪些是底线不能妥协，哪些可以成本取舍。
4. **给出竞品参照物。** "参考 [竞品ASIN]，但在 [具体部位] 需要改成 [具体方案]"。

## 内部堆叠剖面图方法论

### 为什么要在根因拆解后、创新亮点前做这一步？

| 没有堆叠剖面图 | 有堆叠剖面图 |
|---------------|-------------|
| 根因拆解写了「加气液分离挡板」，但不知道放在哪里 | 剖面图上标注「此处加挡板」，一目了然 |
| 工业设计天马行空，画出来的产品内部放不下 | 外观设计严格受内部尺寸约束，每个造型都能落地 |
| 供应商看文字描述，理解 50% | 供应商看带标注的剖面图，理解 95% |
| 设计与工程脱节，打样返工 | 设计即工程，一次到位 |

### 执行流程

#### Step 5.1：调研内部结构

通过以下渠道调研目标品类的内部组件堆叠：

1. **WebSearch 搜索关键词：**
   - `[product type] teardown`
   - `[product type] internal structure diagram`
   - `[product type] disassembly repair`
   - `[product type] patent drawing internal`（Google Patents CPC 分类号）
   - `[product type] OEM component specification`

2. **提取信息：**
   - 所有内部组件列表（名称、材质、尺寸）
   - 组件堆叠顺序（从上到下/从前到后）
   - 关键热区、运动部件、密封面
   - 为什么外形长这样——哪个内部组件决定了外部轮廓

3. **画出堆叠示意图（ASCII）：**
   在蓝图 MD 中用 ASCII 图画出组件堆叠关系，标注关键尺寸。

#### Step 5.2：标注改善点

将 Step 4 根因拆解中确定的所有「设计缺陷 → MVP 推荐改良方案」，逐一标注到堆叠图上：

```
标注格式：
[组件名称] ← ⚠️ 改善: [具体改良内容]

例如：
[密封圈 NBR] ← ⚠️ 改善: 换 EPDM 硅胶，硬度 50±5 Shore A
[蒸汽出口（无挡板）] ← ⚠️ 改善: 加 3 层不锈钢多孔挡板 1-2mm
[配件接口 Push-fit] ← ⚠️ 改善: 改 Snap-Lock 卡扣，保持力 ≥50N
```

#### Step 5.3：生成剖面渲染图

调用 `/allen-product-design`，传入堆叠结构 + 改善点标注，生成 1-2 张带标注的内部剖面渲染图：

```
/allen-product-design

【剖面图生成任务】
产品：[品类名称]
类型：内部剖面图（cross-section / cutaway view）

组件堆叠（从上到下）：
1. [组件A] — [尺寸] — [材质]
2. [组件B] — [尺寸] — [材质]
...

改善点标注（必须在图中清晰标注）：
- 位置 X: [改善内容]
- 位置 Y: [改善内容]
...

渲染要求：
- 半透明外壳，内部组件清晰可见
- 改善点用箭头+文字标注
- 技术插图风格，白色背景
- 标注用英文（供国际供应商阅读）
```

### 输出物

| 文件 | 说明 |
|------|------|
| 蓝图 MD 中的 ASCII 堆叠图 | 文字版堆叠结构 + 改善点标注 |
| 剖面渲染图（1-2 张） | 由 allen-product-design 生成的带标注内部剖面图 |

## 工业设计调用规范（更新版）

### 调用时机

工业设计概念图的生成分两次调用：

| 调用 | 时机 | 目的 |
|------|------|------|
| **第一次（Step 5）** | 根因拆解后 | 生成内部剖面图（带改善点标注） |
| **第二次（Step 10）** | MVP 规格书后 | 生成外观概念图（多风格发散） |

### 第二次调用：外观概念图的执行流程

⚠️ **关键变更：不再直接按 MVP 规格书的配色/风格结论生成固定方案，而是先调研堆叠约束，再根据目标用户年龄做风格发散探索。**

#### Phase A：堆叠约束确认

将 Step 5 的内部堆叠结构作为强约束传递给 allen-product-design：

```
【内部堆叠约束（hard constraints，外观设计不可违反）】
- 机身最小宽度：[X]mm（由 [核心组件] 决定）
- 机身最小高度：[Y]mm（由 [组件堆叠] 决定）
- 手柄角度范围：[A-B]°（由人体工学 + 热隔离决定）
- 手柄直径范围：[C-D]mm（由人手握持决定）
- 外形必须是 [品类基本型描述]（保持品类辨识度）
- 重心约束：[描述]（水箱在 [位置]，影响平衡）
```

#### Phase B：目标用户年龄 → 风格发散

根据 Step 3 用户画像中的**主力用户年龄段**，选择匹配的设计风格进行发散探索：

| 用户年龄段 | 推荐发散风格 | 理由 |
|-----------|-------------|------|
| 18-25 岁 | 可爱(cute)、Y2K、波普(playful)、赛博朋克 | 年轻、社媒分享、个性表达 |
| 25-35 岁 | 极简(minimal)、北欧(Scandi)、有机(organic)、苹果风 | 干净利落、品质感、Instagram 审美 |
| 30-45 岁 | 北欧、无印(MUJI)、B&O风、包豪斯(Bauhaus) | 成熟克制、设计内涵、生活品质 |
| 35-50 岁 | 包豪斯、博朗(Braun)、奢华(luxury)、经典 | 永恒设计、材质感、不跟风 |
| 50+ 岁 | 复古(retro)、经典、简约大方 | 易识别、不花哨、质感 |
| 混合/广泛 | 极简 + 任选 2 个匹配风格 | 覆盖主力 + 做差异化尝试 |

**发散规则：**
1. 至少 3-4 种风格各出 2 张（45°+正面），共 6-8 张产品图
2. 每种风格必须遵守堆叠约束和品类基本型（不可天马行空）
3. 每种风格只做 1 个记忆点（遵守「少即是多」准则）
4. 加上使用场景图 2 张 + 色彩变体图 1 张 + 人体工学对比 1 张

#### Phase C：五条设计准则（硬规则）

所有风格发散必须遵守以下 5 条准则：

1. **少即是多** — 每个方案最多 1 个视觉记忆点，其余保持克制
2. **使用基本型** — 基于品类已有的经典外形（如枪型/壶型），用几何基本体（圆柱/长方体/圆锥）构成，不使用复杂异形曲面
3. **保持品类辨识度** — 消费者一眼能认出这是什么产品（类比电动车仍保留类似格栅的前脸）
4. **有且仅有一个记忆特征** — 消费者必须记住一个东西（一圈金环/一条光窗/一块织物带），这个特征同时也是品牌锚点
5. **遵从人体工学** — 使用时手腕保持中性位，不做反关节运动；手柄尺寸/角度/粗细必须匹配使用姿势

### 必须传递的上下文

调用 `allen-product-design` 时，必须将以下前序阶段的信息**完整传递**作为输入，不可只传一句话产品描述：

```
/allen-product-design

【产品基本信息】
- 品类：[手持蒸汽清洁器 / Handheld Steam Cleaner]
- 目标市场：[US]
- 目标价位：[$39.99-$44.99]
- 对标竞品：[CURMEDI B0DGQ7L856, $43.99, 深灰色]

【目标用户画像（Step 3 输出）】
- 主力用户：[25-45岁女性家庭用户，有小孩和宠物]
- 审美偏好：[清新家居感，马卡龙色系，偏好"好看又好用"的小家电]
- 核心诉求：[日常轻量清洁，chemical-free，安全]
- 购买决策：[TikTok/社媒种草 → Amazon 搜索 → 主图+价格+评分 → 3分钟决策]
- 使用场景：[厨房灶台/浴室瓷砖缝/偶尔车内清洁]

【竞品外观现状（要区别于此）】
- 竞品配色：[深灰/黑色为主，市场 90% 同质]
- 竞品材质：[亮面光滑 PP 塑料，廉价感]
- 竞品形态：[水壶型手枪握把，无状态指示，配件推入式]
- 竞品包装：[棕色瓦楞纸，配件散装]

【我们的差异化设计（MVP 规格书定义）】
外观差异化：
- 配色：[薄荷绿 Pantone 337C + 白色点缀]
- 表面：[哑光微纹理]
- 握柄：[白色硅胶包覆 + 菱形防滑纹]

创新特征（必须在渲染图中体现）：
- LED 三色就绪灯：蓝=加热中 / 绿=可用 / 红=缺水
- 透明水箱窗：侧面 PC 窗口，带 100/200/300ml 刻度
- Snap-Lock 配件接口：橙色释放按钮
- 蒸汽锁定按钮：握柄侧面滑动开关
- 旋锁安全盖：白色，表面刻纹

【痛点改善（不可见但影响外观局部）】
- 气液分离挡板：内部结构，不影响外观
- 密封圈：如有附赠备用圈，需在配件图中体现（薄荷绿色硅胶圈）

【产品尺寸参考】
- 外包装：~28×26×17 cm
- 净重：~1.2kg
- 电源线：9.4ft
- 配件数：12件

【渲染需求】
- 产品主图（白底 45°/正面/侧面）
- 创新细节特写（LED / Snap-Lock / 水箱窗）
- 使用场景（厨房/浴室/汽车）
- 开箱体验
```

### 为什么要传完整上下文？

| 只传一句话 | 传完整上下文 |
|-----------|------------|
| "薄荷绿手持蒸汽清洁器" | 上面完整信息 |
| → AI 凭想象画，可能画错品类形态 | → AI 知道竞品长什么样、我们哪里不同 |
| → 创新特征可能漏掉 | → LED/Snap-Lock/水箱窗都会准确体现 |
| → 和 MVP 规格书脱节 | → 渲染图就是规格书的视觉版 |

### 输出物归属

工业设计图由 `allen-product-design` 生成并保存，MVP Blueprint 在报告中引用图片路径即可，不重复管理图片文件。

## 用户影响力评估模型

### 为什么需要这个模型？

差评改善大部分是**内部改良**（换密封圈材质、加气液分离板），用户在 Listing 页面完全感知不到。花了钱但不影响购买决策。真正影响购买的是用户在**搜索结果页看到主图的 1 秒内**、**点进 Listing 扫五点的 30 秒内**能感知到的差异。

### 三类改良/创新的角色

| 类型 | 角色 | 决策影响时机 |
|------|------|-------------|
| **进攻型**（创新亮点+外观） | 驱动购买："为什么选你" | 浏览 Listing 时 |
| **防守型**（差评改善-可见） | 消除顾虑："不会踩雷" | 阅读五点/A+ 时 |
| **隐形型**（差评改善-不可见） | 保护评价/退货率 | 使用后才知道 |

### 四维度评分标准

| 维度 | 权重 | 1 分 | 3 分 | 5 分 |
|------|------|------|------|------|
| **A. Listing 可感知度** | **35%** | 完全不可见 | A+ 图文可展示 | 主图一眼可见 |
| **B. 购买决策影响力** | **30%** | 无感 | 加分项 | 关键决策因子 |
| **C. 使用体验影响力** | **20%** | 无影响 | 小幅改善 | 消除核心差评 |
| **D. 成本效率** | **15%** | >¥10/件 | ¥3-6/件 | ≤¥1/件 |

加权公式：`总分 = A×0.35 + B×0.30 + C×0.20 + D×0.15`

### 隐形改良的可视化包装

对 A 维度得分 ≤2 的改良，必须提出可视化方案：

| 方法 | 例子 |
|------|------|
| **Listing 命名法** | 气液分离 → "Anti-Spit Technology™" |
| **配件可视化** | EPDM 备用圈做成彩色放在配件包里 |
| **对比 Infographic** | A+ 中左竞品漏水 vs 右我们密封特写 |
| **撞色标记** | 改良部件做不同颜色，拍照可见 |

### 创新亮点发掘方向（通用清单）

| 维度 | 低成本方向 |
|------|-----------|
| 视觉差异 | 配色、哑光/磨砂、透明元素、几何造型 |
| 状态反馈 | LED 指示灯、数显、震动/声音提示 |
| 便携/收纳 | 收纳底座、磁吸归位、折叠设计 |
| 开箱体验 | 彩色包装、EVA 内衬、QR 视频教程 |
| 耗材/复购 | 替换刷头包、清洁液、配件升级包 |
| 场景扩展 | 特定场景配件（汽车/窗户/织物套装） |

### 预算分档输出

必须输出 3 档方案：精简版（MVP 推荐，只选高可感知项）、标准版（+关键差评改善）、完整版（全做）。每档标注总增量成本和毛利率影响。
