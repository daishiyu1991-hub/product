# KANO 增强型 MVP 功能矩阵方法论

## 为什么引入 KANO？

原有分类只看差评率阈值（≥15% → Must, 5-14% → Nice, <5% → Cut），致命盲区：**低差评率的创新功能被误杀**。例如"内置温度探针"差评率2%→旧系统Cut，但竞品好评爆发率18%，实际是兴奋型需求。

## KANO 五类定义（亚马逊语境）

| KANO | 符号 | 通俗解释 | 亚马逊信号 | 典型例子 |
|------|------|---------|-----------|---------|
| 基本型 Must-be | M 🔴 | 没有会被骂，有了不加分 | 高差评，好评不提 | 空气炸锅不冒烟 |
| 期望型 Performance | O 🟡 | 做得越好越满意 | 好评差评都提 | 容量大小 |
| 兴奋型 Attractive | A 🟢 | 没有不抱怨，有了会惊喜 | 低差评，好评爆发 | 内置食谱QR码 |
| 无差异 Indifferent | I ⚪ | 有没有都无所谓 | 都不提 | 高级纸说明书 |
| 反向型 Reverse | R ⛔ | 有了反而扣分 | 差评抱怨功能"存在" | $30强制WiFi控制 |

## KANO 算法分类决策树

从 Phase 2 上游数据算法推导（无法做 KANO 用户问卷）：

```
输入：complaint_rate, positive_mention_rate, influence_score, feature_type

┌─ complaint_rate ≥15% 且 positive_mention_rate <5%
│  → MUST-BE (M) 🔴 — 人人骂它没有，没人夸它有
│
├─ complaint_rate ≥5% 且 positive_mention_rate ≥5%
│  → PERFORMANCE (O) 🟡 — 好评差评都提，线性满意度
│
├─ complaint_rate <5% 且 positive_mention_rate ≥10%
│  → ATTRACTIVE (A) 🟢 — 没人抱怨缺少，有了评价亮眼
│
├─ complaint_rate <5% 且 positive_mention_rate <5% 且 influence_score <2.0
│  → INDIFFERENT (I) ⚪ — 都不提，影响力低
│
├─ ≥3条差评明确抱怨该功能"不该存在/多此一举"
│  → REVERSE (R) ⛔ — 用户嫌功能不该有
│
└─ 其他 → PERFORMANCE (O) 🟡（默认值）
```

## KANO → 优先级修正器

| KANO | 修正系数 | 最终映射 | 逻辑 |
|------|---------|---------|------|
| M 🔴 | 锁定 | → Must-have（无论分多低） | 入场门槛，没有被差评淹死 |
| O 🟡 | ×1.0 | 分≥3.0→Must；2.0-2.99→Nice；<2.0→Nice | 按分数排序 |
| A 🟢 | ×1.15 | 修正后≥2.5→Must；<2.5→Nice | 兴奋型有超额ROI |
| I ⚪ | ×0.5 | → Cut | 没人在意=浪费成本 |
| R ⛔ | ×0.0 | → Cut + ⚠️ 主动移除 | 有害功能 |

> **为什么 A 加成15%？** 兴奋型功能是唯一能在主图/第一屏拉开差距的武器。15% 加成让系统不再误杀高价值创新。

## 数据完整性规则

1. **优先从 Phase 2 提取**：complaint_rate 从主报告痛点统计表读取；positive_mention_rate 从 `01_review_*.csv` 筛选评星≥4的行计算
2. **Reverse 门槛严格**：需 ≥3 条抱怨功能"存在"，不是抱怨"质量差"
3. **Attractive 需实证**：需 ≥5 条好评有惊喜表达
4. **无上游时**：降级为 Sorftime `product_reviews` 采集，标注来源

## KANO 如何纠正旧系统的误判（示例）

| 功能 | 旧系统 | 旧理由 | KANO | KANO理由 | 变化 |
|------|-------|--------|------|---------|------|
| 内置温度探针 | ❌ Cut | 差评2%<5% | ✅ Must | A🟢 好评率18% ×1.15达标 | 拯救核心差异化 |
| WiFi控制 | 🟡 Nice | 部分竞品有 | ❌ Cut+⚠️ | R⛔ 8%抱怨"不需要" | 识别有害功能 |
| 不粘涂层 | ✅ Must | 差评31%≥15% | ✅ Must | M🔴 结论一致 | 理由更清晰 |
