# Amazon Listing 全面优化清单

## 概述

本清单涵盖 Amazon Listing 所有可优化维度，用于每次迭代时系统性检查和改进。每个维度包含具体的检查项、最佳实践标准和常见错误提示。建议在每个迭代周期中选择 1-3 个维度进行深度优化，而非试图同时优化所有维度。

---

## 一、标题（Title）优化清单

### 1.1 关键词研究

- [ ] 使用 Helium 10 / Jungle Scout 等工具提取核心关键词（搜索量 > 1000/月）
- [ ] 收集前 10 名竞品标题中的共同关键词
- [ ] 分析 Brand Analytics "搜索词表现" 报告，获取实际转化关键词
- [ ] 检查 Search Query Performance 仪表盘中的高展示低点击词
- [ ] 通过 Amazon 搜索框自动补全发现长尾关键词
- [ ] 查看竞品的 "Customers also searched for" 区域发现关联词
- [ ] 确认关键词与产品高度相关，避免堆砌不相关的高流量词

### 1.2 标题结构与格式

- [ ] 标题长度控制在 150-200 个字符以内（不超过类目限制）
- [ ] 前 80 个字符包含最核心的 1-2 个关键词（移动端截断点）
- [ ] 标题结构遵循公式：品牌名 + 核心关键词 + 关键属性 + 差异化卖点 + 适用场景/人群
- [ ] 每个重要单词首字母大写（介词、冠词除外）
- [ ] 使用连字符（-）或逗号（,）分隔不同信息模块
- [ ] 不使用全大写或特殊符号（★、♥ 等）
- [ ] 不包含价格、促销信息、主观性评价词（best, amazing）
- [ ] 不重复使用同一关键词

### 1.3 竞品标题分析

- [ ] 记录 BSR Top 10 的标题结构和关键词用法
- [ ] 识别竞品标题中你缺少的有效关键词
- [ ] 分析竞品的差异化卖点表达方式
- [ ] 检查是否有竞品在标题中使用了你未覆盖的使用场景

### 1.4 A/B 测试准备

- [ ] 准备至少 2 个标题版本用于 Manage Experiments 测试
- [ ] 两个版本之间只改变一个主要变量（关键词顺序/差异化卖点/结构）
- [ ] 设定测试周期（建议最少 4 周）和成功指标（CTR、转化率、销量）

## 二、图片（Image）优化清单

### 2.1 主图（Main Image）要求

- [ ] 纯白背景（RGB 255,255,255）
- [ ] 产品占据图片面积 85% 以上
- [ ] 分辨率至少 2000 x 2000 像素（支持缩放功能）
- [ ] 无文字、Logo、水印叠加（品牌 Logo 印在产品上的除外）
- [ ] 产品颜色准确还原，无过度修图
- [ ] 主图角度展示产品最佳形态
- [ ] 检查主图在搜索结果页面中缩略图尺寸下的辨识度
- [ ] 与竞品主图对比，确认差异化和吸引力

### 2.2 副图规划（建议使用全部 6-8 个副图位）

- [ ] **副图 1 — 尺寸/规格图**：标注产品实际尺寸，提供参照物对比
- [ ] **副图 2 — 核心卖点信息图**：用图标+简短文字突出 3-4 个核心卖点
- [ ] **副图 3 — 使用场景图**：展示产品在实际使用场景中的效果
- [ ] **副图 4 — 细节特写图**：展示材质、做工、关键功能细节
- [ ] **副图 5 — 对比/差异图**：与竞品或前代产品的差异化对比
- [ ] **副图 6 — 包装内容图**：展示开箱内容，所有配件一览
- [ ] **副图 7 — 生活方式图**：模特/使用者在生活场景中使用产品
- [ ] **副图 8 — 信任背书图**：认证、测试报告、用户好评截图等

### 2.3 信息图（Infographic）设计标准

- [ ] 每张信息图只传达 1 个核心信息
- [ ] 文字大小在手机端可阅读（字号 ≥ 36pt）
- [ ] 图标风格统一，整体设计与品牌调性一致
- [ ] 使用高对比度配色确保可读性
- [ ] 关键卖点文字使用粗体或高亮色强调
- [ ] 避免文字过多导致视觉拥挤

### 2.4 视频（Video）

- [ ] 上传产品视频（30-60 秒最佳）
- [ ] 视频前 5 秒展示产品核心卖点，吸引注意力
- [ ] 包含产品使用演示、实际效果展示
- [ ] 视频分辨率 1080p 以上
- [ ] 添加字幕（考虑静音播放场景）

## 三、五点描述（Bullet Points）优化清单

### 3.1 结构规范

- [ ] 每个 bullet point 控制在 200-250 个字符（不超过 500）
- [ ] 每个 bullet 以核心卖点的大写关键词短语开头（如：PREMIUM QUALITY MATERIAL）
- [ ] 每个 bullet 聚焦一个独立卖点，不混搭多个话题
- [ ] 5 个 bullet 按重要性从高到低排序

### 3.2 内容规划（5 个 Bullet 的推荐主题）

- [ ] **Bullet 1 — 核心价值主张**：产品最大的差异化卖点，解决什么问题
- [ ] **Bullet 2 — 材质/品质**：材料、制造工艺、耐用性
- [ ] **Bullet 3 — 功能特性**：关键功能的具体描述和用户获益
- [ ] **Bullet 4 — 适用场景/人群**：谁适合用？什么场景下用？
- [ ] **Bullet 5 — 售后保障**：保修、退换、客服支持信息

### 3.3 关键词与可读性

- [ ] 每个 bullet 自然融入 2-3 个相关关键词
- [ ] 使用具体数字和数据增强说服力（例：「支持 10 小时连续使用」）
- [ ] 采用 "特征 + 好处" 写法（例：「采用304不锈钢 — 耐腐蚀，使用寿命延长3倍」）
- [ ] 避免使用行业术语，用消费者能理解的语言
- [ ] 不使用全大写句子（仅开头关键短语大写）
- [ ] 不包含价格、促销、物流信息
- [ ] 不使用 HTML 标签

## 四、A+ Content（图文描述）优化清单

### 4.1 模块选择策略

- [ ] 使用 5-7 个内容模块（充分利用空间但不过度冗长）
- [ ] 首模块使用全宽 Banner 图（品牌视觉冲击力）
- [ ] 包含至少一个图文并排模块（Standard Image & Text）
- [ ] 包含至少一个产品对比表（Standard Comparison Chart）
- [ ] 使用 Standard Four Image & Text 模块展示多个卖点
- [ ] 末尾模块添加品牌故事或使命宣言

### 4.2 品牌故事（Brand Story）

- [ ] 开启 Brand Story 模块（出现在 A+ Content 上方）
- [ ] 品牌故事包含品牌起源、理念、使命
- [ ] 使用品牌一致的视觉风格和色调
- [ ] 添加指向品牌旗舰店（Brand Store）的链接
- [ ] 交叉推荐品牌其他产品（引流）

### 4.3 对比图表

- [ ] 列出品牌下 3-5 个相关产品进行横向对比
- [ ] 对比维度选择能突显当前产品优势的属性
- [ ] 每个对比产品添加可点击的 ASIN 链接
- [ ] 使用清晰的图标（✓/✗）标注功能有无

### 4.4 设计与文案标准

- [ ] 所有图片分辨率满足 A+ 要求（宽度 970px）
- [ ] 图片中的文字在手机端可阅读
- [ ] 文案简洁有力，避免大段落文字
- [ ] 配色与品牌视觉识别体系一致
- [ ] 不包含竞品品牌名称或贬低竞品的内容
- [ ] 不包含保修、退换等售后信息（A+ 政策限制）
- [ ] 所有图片设置了 Alt Text（含关键词，提升 SEO）

### 4.5 Premium A+ Content（如果有资格）

- [ ] 评估是否有 Premium A+ Content 资格（品牌注册+品牌故事完成）
- [ ] 使用交互式模块（Hotspot、Carousel）
- [ ] 添加视频模块
- [ ] 使用更大的宽幅图片

## 五、后台搜索关键词（Backend Keywords）优化清单

### 5.1 关键词填写规则

- [ ] 总字符数不超过 249 bytes（包含空格）
- [ ] 关键词之间用空格分隔，不使用逗号或分号
- [ ] 不重复标题中已有的关键词
- [ ] 不重复 bullet points 中已有的关键词
- [ ] 使用小写字母（大小写不影响索引）
- [ ] 包含常见拼写错误词（自然拼写错误，非故意错误）
- [ ] 包含西班牙语/法语等其他语言的产品名称（如适用）

### 5.2 禁止填写的内容

- [ ] 不包含品牌名称（自己的或竞品的）
- [ ] 不包含 ASIN 编号
- [ ] 不包含侮辱性、攻击性词语
- [ ] 不包含 "best"、"cheapest"、"on sale" 等主观/促销词
- [ ] 不包含临时性描述（如：new, latest）
- [ ] 不重复使用同一词的单数和复数形式（Amazon 会自动匹配）

### 5.3 关键词来源优先级

- [ ] 第一优先：Brand Analytics 高转化但未在前端使用的词
- [ ] 第二优先：广告报告中有转化但未收录的搜索词
- [ ] 第三优先：竞品索引但自己未覆盖的词
- [ ] 第四优先：同义词、缩写词、相关属性词
- [ ] 第五优先：季节性关键词、使用场景词

### 5.4 索引验证

- [ ] 使用 ASIN 反查工具验证关键词是否被 Amazon 索引
- [ ] 对未被索引的关键词重新调整位置（移至标题或 bullet）
- [ ] 每月检查一次索引状态，确保持续有效

## 六、价格（Pricing）优化清单

### 6.1 竞争定价分析

- [ ] 记录 Top 10 竞品的当前售价
- [ ] 计算类目平均价格和价格区间
- [ ] 分析自己产品在价格带中的定位（低价/中价/高价）
- [ ] 评估自己的成本结构和目标利润率
- [ ] 检查价格是否触发了 "选择框被压制" 警告

### 6.2 心理定价策略

- [ ] 使用 .99 或 .97 结尾的价格（例：$24.99 而非 $25.00）
- [ ] 考虑设置划线价格（List Price）制造折扣感
- [ ] 避免定价过低导致消费者质疑品质
- [ ] 关键价格锚点参考：$9.99、$14.99、$19.99、$24.99、$29.99

### 6.3 优惠券与促销

- [ ] 设置 Coupon（搜索结果中显示绿色标签，提高 CTR）
- [ ] Coupon 金额建议：5%-15% 折扣或 $1-$5 固定金额
- [ ] 评估是否设置 Subscribe & Save 折扣
- [ ] 大促期间提前设置 Lightning Deal / Best Deal
- [ ] 检查促销叠加是否会导致利润为负

### 6.4 价格弹性测试

- [ ] 记录当前价格下的日均销量和转化率
- [ ] 小幅调价（±5-10%），观察 7 天销量变化
- [ ] 计算价格弹性系数 = 销量变化% / 价格变化%
- [ ] 基于弹性系数确定利润最大化的最优价格

## 七、品牌内容（Brand Content）优化清单

### 7.1 品牌旗舰店（Brand Store）

- [ ] 创建或更新品牌旗舰店（至少 3 个页面）
- [ ] 首页展示品牌形象、核心产品系列、当季主推
- [ ] 按产品类别创建子页面，方便浏览
- [ ] 添加 "关于我们" 页面讲述品牌故事
- [ ] 使用全宽 Hero 图片和视频提升视觉体验
- [ ] 所有产品 Tile 链接正确，无断链
- [ ] 旗舰店的移动端显示效果检查
- [ ] 设置旗舰店流量来源追踪标签（Store Source Tag）

### 7.2 帖子（Posts）

- [ ] 定期发布品牌帖子（建议每周 3-5 条）
- [ ] 帖子图片高质量，与 Instagram 风格类似
- [ ] 帖子文案简洁有吸引力，包含行动号召
- [ ] 关联相关产品 ASIN
- [ ] 分析帖子数据，优化高互动类型的内容

### 7.3 品牌跟踪（Brand Follow）

- [ ] 在品牌旗舰店中突出 "Follow" 按钮
- [ ] 通过优质内容积累品牌关注者
- [ ] 监控品牌关注者增长趋势

## 八、移动端（Mobile）优化清单

### 8.1 移动端显示检查

- [ ] 在手机上查看 Listing 完整页面，截图记录
- [ ] 标题前 80 字符是否包含核心关键词和差异化信息
- [ ] 主图缩略图在小屏幕上是否清晰可辨
- [ ] 副图中的文字在手机屏幕上是否可阅读
- [ ] A+ Content 的图片和文字在手机端的显示效果
- [ ] 价格和优惠信息是否在首屏可见

### 8.2 移动端特有检查项

- [ ] 检查 "From the brand" 区域的显示（品牌旗舰店内容）
- [ ] 检查 video 在移动端的播放效果
- [ ] 确认 bullet points 展开/折叠后的可读性
- [ ] 检查评论区的照片/视频评论展示
- [ ] 确认 "Compare with similar items" 区域中自己产品的展示

### 8.3 移动端转化优化

- [ ] 确保最重要的信息在手机首屏（无需滚动即可看到）
- [ ] 主图需要在移动端也能传达产品核心价值
- [ ] A+ Content 中避免使用纯文字模块（移动端阅读体验差）
- [ ] 所有 CTA（行动号召）按钮在手机端可轻松点击

## 九、季节性调整清单

### 9.1 季节前准备（提前 4-6 周）

- [ ] 识别即将到来的季节性机会（节日、大促、季节变换）
- [ ] 准备季节性关键词列表（例：Christmas gift, summer outdoor）
- [ ] 设计季节性副图和 A+ Content 图片
- [ ] 规划季节性定价和促销策略
- [ ] 调整广告预算和竞价策略
- [ ] 确保库存充足以应对季节性需求高峰

### 9.2 季节高峰期间

- [ ] 将季节性关键词融入标题和 bullet points
- [ ] 替换副图为季节性版本（如节日主题生活方式图）
- [ ] 更新 A+ Content 突出季节性使用场景
- [ ] 激活预先设置好的优惠券和促销
- [ ] 加大广告投放力度，提高竞价
- [ ] 每日监控库存水平

### 9.3 季节结束后

- [ ] 恢复常规版本的标题和图片
- [ ] 移除季节性关键词
- [ ] 恢复常规价格和广告策略
- [ ] 总结季节性表现数据，为下一年做参考
- [ ] 归档季节性素材以便下次复用
