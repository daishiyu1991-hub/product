# Kill / Continue / Pivot 决策框架

> 本文档为亚马逊产品在上线运营一段时间后，提供一套系统化的"继续投入 / 调整方向 / 果断止损"决策方法论。
> 适用时机：产品上线 30–90 天后，或任何需要评估产品去留的关键节点。

---

## 一、六维红绿灯评估体系

### 1.1 评估维度定义

对产品当前表现进行六个维度的红绿灯评级：

#### 维度 1：转化率（Conversion Rate）

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | 转化率 ≥ 同价位良好基准 | 产品吸引力强，listing质量高 |
| 🟡 黄灯 | 转化率在合格线–良好线之间 | 有优化空间但不紧急 |
| 🔴 红灯 | 转化率 < 合格线，且优化后仍无改善 | 产品/市场存在根本性问题 |

**数据来源**：Business Reports → Detail Page Sales and Traffic → Unit Session Percentage
**评估周期**：至少连续14天数据（排除异常波动日）

#### 维度 2：BSR 趋势（Best Sellers Rank Trend）

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | BSR 周均值连续2周以上下降（排名数字变小） | 销售势头向好 |
| 🟡 黄灯 | BSR 平稳波动（周均变化 ±10% 以内） | 销量稳定但无增长 |
| 🔴 红灯 | BSR 连续2周以上上升（排名数字变大）超过20% | 销量持续下滑 |

**数据来源**：Keepa / Helium10 / Jungle Scout 等工具追踪
**评估周期**：至少4周趋势数据

#### 维度 3：ACOS / TACoS

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | ACOS ≤ 当前阶段目标值，且TACoS呈下降趋势 | 广告效率健康 |
| 🟡 黄灯 | ACOS 超出目标但 < 盈亏平衡ACOS，TACoS平稳 | 广告需优化但尚可接受 |
| 🔴 红灯 | ACOS > 盈亏平衡ACOS，且TACoS持续上升 | 广告烧钱且无法带动自然流量 |

**数据来源**：广告后台 Campaign Manager
**评估周期**：至少2周（排除新campaign学习期数据）

#### 维度 4：退货率（Return Rate）

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | 退货率 < 类目正常值 | 产品质量和描述匹配度好 |
| 🟡 黄灯 | 退货率在正常值–警戒线之间 | 需关注退货原因分布 |
| 🔴 红灯 | 退货率 > 警戒线，或连续上升 | 产品或listing存在严重问题 |

**数据来源**：Reports → Return Reports
**评估周期**：累计至少50个订单后评估才有统计意义

#### 维度 5：评分与评论（Rating & Reviews）

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | 平均评分 ≥ 4.2，且近30天无下降趋势 | 产品口碑良好 |
| 🟡 黄灯 | 平均评分 3.8–4.2，或近期出现几条差评 | 需要关注差评内容并改进 |
| 🔴 红灯 | 平均评分 < 3.8，或出现系统性差评问题 | 产品存在质量/功能硬伤 |

**数据来源**：产品详情页 + Review分析工具
**评估周期**：至少积累10条以上评论后评估

#### 维度 6：自然流量占比（Organic Traffic Share）

| 信号 | 判断标准 | 说明 |
|------|---------|------|
| 🟢 绿灯 | 自然流量占比 ≥ 对应月份基准值 | 产品正在建立自然流量根基 |
| 🟡 黄灯 | 自然流量占比略低于基准（差距<10个百分点） | 关键词排名提升偏慢 |
| 🔴 红灯 | 自然流量占比远低于基准（差距>15个百分点） | 产品对关键词排名几乎没有贡献 |

**数据来源**：Brand Analytics + Search Query Performance
**评估周期**：按月评估

### 1.2 综合评分规则

将六个维度的红绿灯汇总，按以下规则进行决策：

```
┌─────────────────────────────────────────────────────┐
│                  决策矩阵                             │
├─────────────────────────────────────────────────────┤
│                                                      │
│  ✅ CONTINUE（继续推进）                              │
│     - 绿灯 ≥ 4个，且无红灯                            │
│     - 绿灯 ≥ 5个，且红灯 ≤ 1个（非一票否决维度）        │
│                                                      │
│  🔄 PIVOT（调整方向）                                  │
│     - 绿灯 2–3个 + 红灯 1–2个                         │
│     - 绿灯 ≥ 3个 + 红灯 1个（属于可改善的维度）         │
│     - 存在明确的改进方向且成本可控                      │
│                                                      │
│  ❌ KILL（停止投入）                                   │
│     - 红灯 ≥ 3个                                     │
│     - 绿灯 ≤ 1个                                     │
│     - 触发任一"一票否决"条件                           │
│     - 连续2个评估周期为 PIVOT 但无改善                  │
│                                                      │
└─────────────────────────────────────────────────────┘
```

## 二、一票否决条件（Veto Rules）

以下情况无论其他维度表现如何，**直接触发 KILL 或强制 PIVOT**：

### 2.1 强制 KILL 条件

| 编号 | 条件 | 原因 |
|------|------|------|
| V1 | 产品被亚马逊下架/限制销售，且无法申诉恢复 | 平台层面终止 |
| V2 | 评分跌至 3.0 以下且持续 30 天无法回升 | 口碑崩塌不可逆 |
| V3 | 退货率超过危险线（类目上限）连续 4 周 | 产品存在根本缺陷 |
| V4 | 收到侵权投诉/安全投诉且证据确凿 | 法律与合规风险 |
| V5 | 累计亏损已超过初始预算的 150%，且无任何改善迹象 | 资金保护 |
| V6 | 类目出现颠覆性竞品（价格低50%+且功能相当） | 市场结构性变化 |

### 2.2 强制 PIVOT 条件

| 编号 | 条件 | 原因 |
|------|------|------|
| P1 | 转化率持续低于合格线 30天，但退货率和评分健康 | 定位/定价问题，非产品问题 |
| P2 | ACOS 无法降到盈亏平衡线以下，但自然流量占比在增长 | 关键词策略需要调整 |
| P3 | 某一特定退货原因占比超过 50% 且可通过改进解决 | 产品需要迭代 |
| P4 | 评论中反复出现同一负面反馈但产品整体评分尚可 | 有明确的改进方向 |

## 三、PIVOT（调整方向）详解

### 3.1 Pivot 的定义

Pivot 不是"放弃"，而是**在保留核心资产（供应链、listing、评论）的基础上，改变产品的某个关键变量**来寻求更好的产品-市场匹配（Product-Market Fit）。

### 3.2 四种 Pivot 类型

#### 类型 A：价格 Pivot（Price Pivot）

**定义**：调整售价和/或成本结构，改变产品的价值定位。

**适用场景**：
- 产品质量好（评分高、退货低）但转化率低
- 竞品价格带发生变化
- 目标客户群的支付意愿判断偏差

**具体操作**：
```
方案1：降价渗透
  - 保持产品不变，降价10%–25%进入更低价格带
  - 风险：毛利被压缩，需要更高销量来弥补
  - 案例：某硅胶厨具从$24.99降至$18.99，转化率从6%提升至14%

方案2：降本降价
  - 简化非核心功能/配件，降低成本同时降低售价
  - 风险：可能影响评论中的感知价值
  - 案例：某收纳盒去掉标签贴纸配件，成本降$1.5，售价降$3

方案3：升级提价
  - 增加高感知价值的功能/配件，提价进入更高价格带
  - 风险：需要额外开发投入
  - 案例：某手机支架增加无线充电功能，售价从$19.99提至$34.99
```

#### 类型 B：功能 / 产品 Pivot（Feature Pivot）

**定义**：修改产品的物理特征、材质、规格或配件，解决用户的痛点。

**适用场景**：
- 评论中反复出现可改进的问题
- 退货原因集中在某个可修复的缺陷
- 竞品分析发现差异化机会

**具体操作**：
```
步骤1：从差评中提取 Top 3 可改进点
步骤2：评估每个改进点的成本和可行性
步骤3：选择"投入产出比最高"的改进点优先执行
步骤4：创建新的子 ASIN 或 Variation 测试改进版本

案例：
  某宠物饮水器差评集中在"水泵噪音大"和"不好拆洗"
  → 更换静音水泵（成本+$0.80）+ 改为可拆卸设计（模具费$2000）
  → 改进版评分从 3.9 提升至 4.5，转化率提升 40%
```

#### 类型 C：受众 Pivot（Audience Pivot）

**定义**：保持产品基本不变，改变目标用户群体。

**适用场景**：
- 产品在预期类目表现不佳，但在意外的类目有机会
- 发现非预期的用户群体对产品有兴趣
- 搜索词报告显示大量非目标关键词带来转化

**具体操作**：
```
步骤1：分析 Search Term Report 中的高转化非目标关键词
步骤2：分析竞品的关联产品，发现交叉销售机会
步骤3：调整 listing 的关键词、图片、卖点以匹配新受众
步骤4：调整广告投放关键词和 ASIN 定向

案例：
  某硅胶冰格原定位"家庭厨房用品"，转化率5%
  → 发现 "whiskey ice mold" 关键词转化率18%
  → 重新定位为"威士忌配件"，更换主图为酒杯场景
  → 整体转化率提升至13%，ACOS从55%降至22%
```

#### 类型 D：定位 Pivot（Positioning Pivot）

**定义**：保持产品和受众不变，改变品牌故事、视觉表达和价值主张。

**适用场景**：
- 产品功能ok但listing没有传达出核心价值
- 竟品的listing质量明显更高
- A/B测试显示不同卖点有显著转化差异

**具体操作**：
```
步骤1：研究类目 Top 10 的listing共性和差异点
步骤2：识别买家最在意的3个决策因素
步骤3：重新设计主图、五点描述和A+页面
步骤4：通过 Manage Your Experiments 进行 A/B 测试

案例：
  某便携风扇原始定位"桌面小风扇"，强调"风力大"
  → 分析发现买家更在意"静音"和"电池续航"
  → 重新定位为"办公室静音便携风扇"
  → 主图增加分贝标注+续航时间标注
  → 转化率从7%提升至12%
```

### 3.3 Pivot 决策检查表

在执行 Pivot 之前，确认以下问题：

```
□ 是否有明确的数据支撑 Pivot 方向？（不能靠直觉）
□ Pivot 的预计成本是多少？（包括时间、资金、机会成本）
□ Pivot 后的预期改善幅度是多少？（需要量化）
□ 最快可以看到效果的时间是多久？
□ 如果 Pivot 也失败了，下一步是什么？（通常是 Kill）
□ 当前的库存如何处理？（是否需要先清库存再重新上架）
□ Pivot 是否可以通过 Variation 测试而非全面替换？
```

## 四、KILL（止损退出）流程

### 4.1 Kill 决策确认

当决策指向 Kill 时，执行以下确认步骤：

```
Kill 确认清单：
  □ 已经进行了至少2轮优化但无改善
  □ 已经考虑了所有 Pivot 可能性并排除
  □ 继续投入的边际回报为负
  □ 团队成员（如有）对 Kill 决策达成共识
  □ 已记录导致 Kill 的核心原因（用于避免重蹈覆辙）
```

### 4.2 Kill 执行步骤

#### 阶段一：停止出血（第1–3天）

```
1. 立即停止所有广告活动（暂停而非归档，以保留数据）
2. 停止一切付费推广和站外引流
3. 取消待生产的订单（如有）
4. 通知供应商暂停后续排产
```

#### 阶段二：库存清理（第4–30天）

```
清理策略（按优先级选择）：

策略A：站内降价清仓
  - 优点：最简单，速度快
  - 方法：降价至成本价附近 + 设置大额Coupon
  - 适用：库存量 < 500件

策略B：站内 Outlet Deal
  - 优点：有专门的清仓入口流量
  - 方法：提交 Outlet Deal 申请
  - 适用：评分 ≥ 3.0 的产品

策略C：多渠道清仓
  - 优点：分散风险
  - 方法：同时在 eBay / Walmart / 社交媒体清仓群出售
  - 适用：有多平台运营能力

策略D：FBA 移除 + 批量清算
  - 优点：快速清空仓储费
  - 方法：创建 Removal Order → 发送至清算公司
  - 适用：产品价值低，不值得单件销售

策略E：FBA Liquidation（亚马逊清算计划）
  - 优点：零操作成本
  - 方法：在后台提交 Liquidation 申请
  - 回收率：约为售价的 5%–10%
  - 适用：最后手段
```

#### 阶段三：教训总结（清仓期间同步进行）

```
必须记录的复盘内容：

1. 产品失败的根本原因分析
   - 市场判断失误？（需求不足/竞争过激）
   - 产品问题？（质量/功能/设计缺陷）
   - 运营问题？（listing/广告/定价失误）
   - 时机问题？（季节性/趋势变化）

2. 关键数据记录
   - 总投入金额（产品+广告+运营）
   - 总回收金额
   - 净亏损金额
   - 关键指标最终值（转化率/ACOS/评分/退货率）

3. 可复用的资产
   - 供应商关系（是否可用于其他产品）
   - 关键词研究数据
   - 竞品分析报告
   - 包装设计文件

4. 避免重复犯错的教训
   - 下次选品时需要特别关注什么？
   - 测试阶段应该在何时做出止损判断？
   - 哪些预警信号被忽视了？
```

### 4.3 Kill 后的仓储费提醒

```
⚠️ FBA 仓储费提醒：

  月度仓储费：
    标准尺寸：$0.87/立方英尺（1–9月）/ $2.40/立方英尺（10–12月）
    大件：$0.56/立方英尺（1–9月）/ $1.40/立方英尺（10–12月）

  库龄附加费（Aged Inventory Surcharge）：
    271–365天：$1.50/立方英尺
    365天+：$6.90/立方英尺 或 $0.15/件（取较高者）

  → 快速清仓的时间窗口通常在决定 Kill 后的 2–4 周内
  → 超过这个窗口，仓储费会显著侵蚀残余价值
```

## 五、CONTINUE（继续推进）流程

### 5.1 Continue 确认后的下一步

当决策为 Continue 时，需要明确当前阶段和下一阶段的行动计划：

```
Continue 不等于"什么都不做"。Continue 意味着：
  1. 当前策略方向正确，继续执行
  2. 针对黄灯指标制定改进计划
  3. 设定下一个评估节点
  4. 加大在有效渠道的投入
```

### 5.2 Continue 行动清单

#### 如果处于上线期（1–4周）→ 重点是冲排名

```
□ 广告预算是否已充分释放？（日预算是否经常被耗尽）
□ 核心关键词自然排名是否在提升？
□ Vine 评论是否已到位？
□ 是否需要配合 Coupon / Deal 加速冲量？
□ 库存是否充足（至少60天库存）？
```

#### 如果处于成长期（5–12周）→ 重点是优化效率

```
□ 否定无效关键词（Search Term Report 分析）
□ 提取高转化词作为精准匹配单独打
□ 优化 listing 的转化元素（A/B 测试主图/标题）
□ 评论数是否在持续增长？
□ TACoS 是否在下降趋势？
```

#### 如果处于成熟期（13周+）→ 重点是利润最大化

```
□ 逐步降低广告出价，测试利润最优点
□ 考虑品牌广告（SB）和展示广告（SD）组合
□ 是否有拓展 Variation 的机会？
□ 是否可以提价？（通过A/B测试验证）
□ 防御性策略：品牌关键词保护、竞品 ASIN 投放
```

### 5.3 设定下一个评估节点

```
无论 Continue 的执行多顺利，都必须设定下一个评估节点：

  上线期 → 每周评估一次
  成长期 → 每2周评估一次
  成熟期 → 每月评估一次

  每次评估使用同样的六维红绿灯体系
  连续3次评估均为 Continue → 可延长评估间隔
  任何一次评估出现 ≥2个红灯 → 立即启动 Pivot 评估
```

## 六、案例分析

### 案例 1：KILL 决策 —— 竹纤维洗碗布

```
产品：竹纤维洗碗布 3件装，售价 $12.99
上线时间：10周
累计投入：$8,500（产品$4,000 + 广告$3,500 + 运营$1,000）

六维评估：
  转化率：5.2%（该价位合格线12%）→ 🔴
  BSR趋势：从 #15,000 上升至 #45,000 → 🔴
  ACOS：78%（盈亏平衡ACOS 35%）→ 🔴
  退货率：8%（类目正常3%–6%）→ 🟡
  评分：3.6（12条评论）→ 🔴
  自然流量占比：8%（10周基准30%）→ 🔴

结果：5个红灯 + 1个黄灯 → 果断 KILL

根因分析：
  - 市场高度同质化，头部竞品有300+评论，价格更低
  - 产品无差异化卖点
  - 差评集中在"掉纤维""不吸水"——产品质量问题

教训：
  - 选品阶段对竞争强度估计不足
  - 样品测试不够充分，未发现掉纤维问题
  - 价格带已被头部玩家锁死，新卖家无利润空间
```

## 七、沉没成本偏差警告

### 7.1 什么是沉没成本偏差

```
⚠️ 沉没成本偏差（Sunk Cost Fallacy）

  "我已经投了这么多钱了，不能就这样放弃。"
  "再坚持一下也许就好了。"
  "如果现在停下来，之前的投入就全白费了。"

  以上想法都是沉没成本偏差的典型表现。

  事实是：已经花出去的钱无论你做什么决策都无法收回。
  正确的思考方式是：从今天起，继续投入能否带来正回报？
```

### 7.2 对抗沉没成本偏差的方法

```
方法1：归零思考法
  问自己：如果今天是第一天看到这个产品（不知道之前投了多少钱），
  我是否会选择投入推广它？
  如果答案是"不会"→ KILL

方法2：机会成本法
  问自己：如果把接下来要投入的资金和时间，用在一个新产品上，
  预期回报是否更高？
  如果答案是"是" → KILL

方法3：设定止损线
  在产品上线前就设定明确的止损线：
  - 最大可接受总投入金额
  - 最晚评估时间点
  - 必须达到的最低指标
  到达止损线时严格执行，不给自己找借口

方法4：第三方视角
  把产品数据给一个不了解项目历史的人看，
  让他/她做出判断。没有情感包袱的人更容易做出理性决策。
```

### 7.3 常见的自我欺骗话术

| 自我欺骗话术 | 理性回应 |
|-------------|---------|
| "再给它一个月" | 一个月后会有什么结构性变化？如果没有，一个月后你还是会说这句话 |
| "竞品也不赚钱" | 竞品不赚钱不能证明你能赚钱 |
| "等评论多了就好了" | 如果产品有问题，评论多了评分只会更低 |
| "旺季就会好起来" | 旺季竞品也在发力，你的相对优势不会自动提升 |
| "已经投了XX万了" | 这XX万已经花了，和未来的决策无关 |
| "我的产品比竞品好" | 数据说了什么？买家用钱投票了什么？ |

---

> **总结**：Kill/Continue/Pivot 决策的核心原则是 **"用数据说话，不用情感决策"**。
> 每一次评估都要回到六维红绿灯体系，用客观指标而非主观感受来驱动决策。
> 记住：及时 Kill 一个失败的产品，释放出来的资源可以用于打造下一个成功的产品。
