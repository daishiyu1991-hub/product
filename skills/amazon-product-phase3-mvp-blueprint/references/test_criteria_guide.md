# 测试标准定义指南（各品类基准值）

## 概述

本指南提供 MVP 产品上架后的量化测试标准，覆盖各主要品类的 KPI 基准值、计算方法和决策规则。所有测试标准必须量化为 Pass/Fail 阈值，禁止"表现良好"等非量化描述。

## 2. 关键指标计算方法

### 2.1 ACOS 目标值计算

**公式**：

```
目标 ACOS = 毛利率 × 分配系数

分配系数选择：
- 保守策略（推荐新手）：50%
- 中等策略（有经验卖家）：60%
- 激进策略（抢占市场份额）：70%
```

**计算示例**：

| 场景 | 售价 | 单位总成本 | 毛利率 | 保守 ACOS | 中等 ACOS | 激进 ACOS |
|------|------|-----------|--------|-----------|-----------|-----------|
| 充电宝 | $25.99 | $15.60 | 40% | ≤20% | ≤24% | ≤28% |
| 香薰机 | $29.99 | $16.50 | 45% | ≤22.5% | ≤27% | ≤31.5% |
| 研磨器 | $19.99 | $10.00 | 50% | ≤25% | ≤30% | ≤35% |

**注意事项**：
- 新品期前 14 天 ACOS 可放宽至目标值 ×1.5（用于冲排名）
- 30 天后 ACOS 必须回落至目标范围内
- 如果 ACOS 持续 >毛利率（即广告亏损），需立即优化或暂停

### 2.2 BSR 目标值计算

**计算方法**：

```
Step 1: 获取子类目 Top 100 竞品的 BSR 分布
Step 2: 计算分位值
  - 30 天目标 = Top 100 中 P50（中位数）排名
  - 60 天目标 = Top 100 中 P30 排名
  - 90 天目标 = Top 100 中 P20 排名

Step 3: 换算日均销量
  BSR → 日均销量的经验公式（美国站参考）：
  - BSR 1~100（大类目）：≈ 300~50 件/天
  - BSR 100~500：≈ 50~15 件/天
  - BSR 500~2000：≈ 15~5 件/天
  - BSR 2000~5000：≈ 5~2 件/天
  - BSR 5000~10000：≈ 2~1 件/天
```

**实操建议**：
- 使用 `product_detail` 获取竞品 BSR，取子类目排名（非大类目）
- 如有 `product_trend` 工具，优先使用 30 天 BSR 均值而非单日快照
- 不同子类目 BSR 差异巨大，必须按实际子类目计算

### 2.3 转化率基准值（按价格区间）

| 价格区间（USD） | 低转化率预警 | 正常范围 | 优秀水平 | 说明 |
|-----------------|------------|---------|---------|------|
| $0~$15 | <8% | 10~18% | >18% | 低价冲动购买，转化率高 |
| $15~$30 | <6% | 8~15% | >15% | 中价位，需要一定说服力 |
| $30~$50 | <5% | 7~12% | >12% | 中高价位，决策更慎重 |
| $50~$100 | <4% | 5~10% | >10% | 高价位，需要强信任背书 |
| $100+ | <3% | 4~8% | >8% | 高客单价，考虑期长 |

**影响转化率的核心因素（按权重排序）**：
1. 主图质量（30%）— 点击后"第一印象"
2. 价格竞争力（25%）— 与搜索结果竞品对比
3. Review 数量和评分（20%）— 社会证明
4. 五点描述质量（15%）— 解决购买疑虑
5. A+ Content（10%）— 品牌故事和深度说服

## 3. 测试周期与里程碑

### 3.1 30/60/90 天评估框架

#### 第 30 天里程碑（首次全面评估）

| 评估维度 | 检查项 | Pass 条件 | 数据来源 |
|----------|--------|-----------|----------|
| 流量 | 广告曝光量 | ≥10,000 次曝光 | 广告报表 |
| 流量 | 自然搜索排名 | 核心词进入前 3 页 | Sorftime / 手动搜索 |
| 转化 | Session 转化率 | ≥品类基准值 | Seller Central |
| 转化 | 广告转化率 | ≥品类基准值×0.8 | 广告报表 |
| 财务 | ACOS | ≤目标值×1.3（新品宽限） | 广告报表 |
| 财务 | 日均销量 | ≥盈亏平衡销量×0.8 | Seller Central |
| 产品 | 退货率 | ≤品类 Fail 阈值 | Seller Central |
| 产品 | 平均评分 | ≥3.8（首批 Review） | 产品页面 |

**第 30 天决策矩阵**：

| 场景 | 核心指标表现 | 决策 | 行动 |
|------|-------------|------|------|
| A | 全部 Pass | **Continue** | 加大广告预算，准备第二批补货 |
| B | 转化率 Pass + ACOS 观察区 | **Continue + 优化** | 优化广告结构，降低无效花费 |
| C | 转化率观察区 + 其他正常 | **优化观察** | 优化 Listing（主图/标题/五点），再观察 14 天 |
| D | 任一核心指标 Fail | **Pivot 或 Kill** | 深度分析原因，14 天内无改善则 Kill |

#### 第 60 天里程碑（中期确认）

| 评估维度 | 检查项 | Pass 条件 | 说明 |
|----------|--------|-----------|------|
| 趋势 | 销量趋势 | 周环比 ≥0%（不下滑） | 排除促销日异常 |
| 财务 | ACOS | ≤目标值（不再宽限） | 60 天后必须达标 |
| 财务 | 累计利润 | 正值或亏损收窄趋势 | 看趋势比看绝对值重要 |
| 产品 | Review 总数 | ≥10 条 | Vine + 自然评论 |
| 产品 | 退货原因 | 无重复性产品质量问题 | 分析退货报告 |

#### 第 90 天里程碑（终极评估）

| 评估维度 | 检查项 | Pass 条件 | Fail 则 Kill |
|----------|--------|-----------|-------------|
| 财务 | 月度净利润 | 正值（扣除广告） | 连续亏损 |
| 财务 | ACOS | ≤目标值 | 持续超标 |
| 排名 | BSR | 稳定在目标范围 | 持续下滑 |
| 产品 | 评分 | ≥4.0 | <3.5 且无有效改善方案 |
| 产品 | 退货率 | ≤品类均值 | 超品类均值 2 倍 |
| 竞争 | 市场份额 | 有稳定的自然流量占比 ≥30% | 完全依赖广告流量 |

### 3.2 每周监控指标（自动化看板）

| 指标 | 监控频率 | 预警阈值 | 说明 |
|------|----------|----------|------|
| 日均销量 | 每日 | 连续 3 天 <盈亏平衡销量 | 排除非工作日影响 |
| ACOS | 每周 | 周 ACOS > 目标值×1.5 | 需立即检查广告 |
| 转化率 | 每周 | 周转化率 < Fail 阈值 | 可能是 Listing 被改或差评 |
| 退货 | 每周 | 周退货 >5 件 | 检查退货原因 |
| 库存 | 每周 | 可售天数 <21 天 | 触发补货流程 |

## 4. Pass/Fail 判定模板

### 4.1 综合评估表模板

```markdown
# MVP 测试评估报告 — [产品名称]

## 基本信息
- ASIN: [ASIN]
- 上架日期: [日期]
- 评估日期: [日期]（第 [N] 天）
- 评估人: [姓名]

## 核心指标评估

| # | 指标 | 目标值 | 实际值 | 判定 | 备注 |
|---|------|--------|--------|------|------|
| 1 | 转化率 | ≥[X]% | [X]% | ✅Pass / ⚠️观察 / ❌Fail | |
| 2 | BSR 排名 | Top [N] | #[N] | ✅Pass / ⚠️观察 / ❌Fail | |
| 3 | ACOS | ≤[X]% | [X]% | ✅Pass / ⚠️观察 / ❌Fail | |
| 4 | 退货率 | ≤[X]% | [X]% | ✅Pass / ⚠️观察 / ❌Fail | |
| 5 | 产品评分 | ≥[X] | [X] | ✅Pass / ⚠️观察 / ❌Fail | |
| 6 | 日均销量 | ≥[N] 件 | [N] 件 | ✅Pass / ⚠️观察 / ❌Fail | |

## 综合判定

- [ ] **Continue** — 全部 Pass 或多数 Pass + 少数观察区
- [ ] **Continue + 优化** — 多数 Pass + 部分观察区
- [ ] **Pivot** — 多数观察区 + 少数 Fail，有明确优化方向
- [ ] **Kill** — 多数 Fail 或触发红线指标

## 下一步行动
1. [行动项1]
2. [行动项2]
3. [行动项3]
```

## 5. 红线指标（触发立即 Kill 决策）

以下任何一项触发 = 立即停止广告投入，评估是否清仓退出：

| # | 红线指标 | 触发条件 | 原因 | 紧急行动 |
|---|----------|----------|------|----------|
| 1 | ⛔ 安全/合规投诉 | 收到任何安全相关投诉或 A-to-Z 投诉 | 可能导致 Listing 下架、店铺受限 | 立即下架产品，排查问题根源 |
| 2 | ⛔ 高退货率 | 退货率 >25%（连续 14 天） | 产品存在系统性质量问题 | 暂停广告，分析退货原因报告 |
| 3 | ⛔ 灾难性评分 | 评分 <3.0（累计 10+ 条 Review） | 几乎不可能逆转至 4.0+ | 评估清仓或重新上架（改良版） |
| 4 | ⛔ 侵权投诉 | 收到知识产权侵权投诉（专利/商标/版权） | 法律风险，可能导致店铺关闭 | 立即下架，法务评估 |
| 5 | ⛔ ACOS 失控 | ACOS >100%（连续 14 天，非新品冲刺期） | 广告费超过售价，持续烧钱 | 暂停所有广告，重新评估关键词和竞价策略 |
| 6 | ⛔ 零转化 | 连续 7 天有点击无转化（点击 >50 次） | Listing 存在根本性问题 | 暂停广告，全面重做 Listing |
| 7 | ⛔ 库存积压预警 | 库存周转天数 >180 天 | 资金被严重占用 | 启动促销清仓，回收现金流 |
| 8 | ⛔ 品类政策变更 | 亚马逊调整品类政策导致无法销售 | 不可抗力 | 立即评估影响范围，决定退出或合规整改 |

### Kill 决策确认流程

```
红线触发
  │
  ├─ Step 1: 确认数据准确性（排除数据异常/系统 Bug）
  │
  ├─ Step 2: 分析根本原因
  │     ├─ 产品本身问题 → Kill（改良后可重新上架）
  │     ├─ Listing 问题 → Pivot（优化后再测 14 天）
  │     └─ 外部因素（季节/竞争） → 观察（暂停投入，等待时机）
  │
  ├─ Step 3: Kill 执行
  │     ├─ 暂停所有广告
  │     ├─ 计算当前库存清仓成本
  │     ├─ 选择清仓方式（降价/Outlet/销毁）
  │     └─ 记录失败原因（供后续选品参考）
  │
  └─ Step 4: 复盘文档归档
```

## 6. 特殊场景处理

### 6.1 季节性品类调整

| 品类特征 | 测试时机建议 | 指标调整 |
|----------|-------------|----------|
| 强季节性（泳装/圣诞装饰） | 旺季前 60 天上架 | BSR 目标放宽 30%，以旺季数据为主要判断 |
| 弱季节性（风扇/加湿器） | 季节转换前 30 天上架 | 允许淡季 ACOS 放宽至目标值×1.3 |
| 无季节性（手机壳/厨具） | 随时上架 | 标准指标，无需调整 |

### 6.2 高价 vs 低价品调整

| 价格段 | 转化率倍数 | ACOS 调整 | 测试期建议 |
|--------|-----------|-----------|-----------|
| <$15 | 基准×1.2 | 标准 | 30 天 |
| $15~$50 | 基准×1.0 | 标准 | 30 天 |
| $50~$100 | 基准×0.8 | 放宽 10% | 45 天 |
| >$100 | 基准×0.6 | 放宽 20% | 60 天 |

### 6.3 多变体产品

- 以主推变体（Hero SKU）的数据为主要判断依据
- 其他变体 BSR 和转化率 Pass 阈值可下调 20%
- 如果主推变体 Fail，不因其他变体 Pass 而保留
- 颜色变体参考：黑/白/灰通常表现最稳定，新品期优先测试
